## One-paragraph summary

We are testing whether the JEPA world-model recipe (encode observations to latents, predict next latents under actions, SIGReg anti-collapse, plan by latent distance to goal) transfers from vision/robotics to **text observations and discrete shell commands**, using a synthetic filesystem sandbox with ground-truth access. Phase 0 (environment + data generation) is complete and has survived two code-review rounds. Phase 1 (world model + probing) is built and mid-experimentation. Standing results: **no recipe variant transfers state to held-out layouts** — training scale, inverse dynamics, data diversity, and set-structured slot latents are all exhausted, and at artifact scale trained encoders decode held-out layouts *worse than random features* (finding 13). Two dissociations sharpen the negative: action-level *delta* information transfers (~9× chance) while absolute state doesn't, and the slow-feature trap is *recipe-dependent* — the CLS recipe suppresses the banner distractor, while the slot recipe's stronger variance demand captures it at 19× chance (finding 12), matching 2211.10831's prediction. The codebase survived an implementation-fidelity audit plus adversarial review (both found and fixed silent recipe deviations that would have confounded these verdicts). **Project goal (restated 2026-07-10): build a terminal agent using JEPA that works.** The diagnosis converges on a theory — prediction+anti-collapse neither pays for nor identifies an absolute-state code in this domain; successful JEPA systems smuggle the missing constraint in via masking, augmentation-invariance, frozen pretrained features, or dense dynamics, and our recipe had none of the four. Next steps ("Path to a working agent", below) add those ingredients in order of expected value. **Track-A status after the 2026-07-13 adversarial review (finding 19): no gate has passed its pre-registered rule.** The comparative results stand — frozen ModernBERT features decode held-out state perfectly under path-keyed readout, and mean-pooled they beat every from-scratch arm by ~16 points (finding 14, corrected) — but an architecture-matched random-init encoder scores identically under that readout except on cwd, so the +20-over-matched-floor clause fails where measurable and the gate-1 PASS is rescinded as a rule verdict. The gate-2 codebook pipeline measures near-oracle but validates decoder + symbolic dynamics, not the A1 arm as pre-registered, and its "rule passed" claim fails the rule's invalid clause under strict scoring (findings 16–17, corrected). The live front was the corrected ladder: hardened referee → pretrained-initialized A1 gate-2 predictor (with a matched random-init control) → gate-1 re-evaluation. **This front is now resolved (2026-07-14, findings 21–22, adversarially reviewed): the recipe-faithful A1 gate-2 arm fails the dynamics-gate calibration clauses (pretraining verdict-irrelevant vs a random-init control), and gate 1 fails its +20/+10 rule against the full-budget architecture-matched floor — so A1 has no cleared gate, and finding-17's gate-2 measurement is confirmed to rest on the symbolic decode bottleneck rather than learned dynamics over frozen features. Both are single-training-seed diagnostics (§7's 5-training-seed bar still open). **The faithful generative-LLM-predictor arm was then built and run (finding 23, 2026-07-15): a pretrained SmolLM2-360M (decoder-only generative LLM, bidirectional) over the frozen features — the VL-JEPA model *type* round 4 lacked. Pretraining now genuinely helps (unlike round 4), but the arm still fails the dynamics gate (no-op/invalid haze, AUC 0.803 < 0.9) and collapses toward copy on held-out layouts. So the frozen-feature *interface* is the wall, not the predictor's model family or scale (within tested range) — strengthening the finding-21/17 conclusion that the symbolic decode bottleneck carries the codebook pipeline. Next unblocked levers: A2 (grounded temporally-masked prediction) and A3 (state-equivalence invariance), both untested for any feature-space arm; a token-gated larger LLM (Llama-3.2-1B) would test scale; and the owed items — 5-training-seed confirmation and genllm edit-weight-10.**

## Path to a working agent (active plan, 2026-07-10)

The product constraint that shapes everything: **a deployed terminal agent meets a new sandbox every session — every real episode is a held-out layout** — so cross-layout state transfer is not an academic bar, it is the product requirement. The theory (findings 11–13 + the identifiability analysis) says our recipe lacked every mechanism that forces content into latents; the plan adds them, cheapest-diagnostic-first, in two parallel tracks.

**Success is defined behaviorally, not by probes** (review correction, adopted): each Track A arm climbs a gate ladder, and only the last rung counts as product progress —
1. *Representation gate* (the original pre-registered margins, not the weakened ones): held-out linear macro ≥ architecture-matched untrained floor **+20** and ≥ reconstruction twin **+10** (protocol v2), with held-out **predicate-satisfaction and exact-state accuracy** reported alongside as the absolute metrics a planner actually needs.
2. *Dynamics gate*: learned predictor over the representation passes rollout-error-by-transition-type (state-changing vs no-op vs invalid — the current no-op miscalibration is a known failure to beat), invalid/no-op calibration, and **goal-distance ranking** (not monotonicity, which valid plans with neutral/setup steps legitimately violate): along scripted optimal plans, the optimal suffix must outrank random and invalid continuations by latent-distance margin, and final distance must improve over the horizon; this is where the violation-of-expectation eval finally gets implemented.
3. *Behavior gate* (the real one, with the §7 bar ported): **held-out-layout goal-reaching** — CEM success ≥ random shooting **+30 points** at matched sampling budget, unfiltered-vs-unfiltered as the primary pairing with filtered-vs-filtered reported, ≥200 episodes over held-out predicates *and* held-out layouts, parser+BFS reported as the nonprivileged ceiling.

**Track A — get a state representation that transfers.** Ordered by expected value:
- **A1 — frozen pretrained text encoder + learned dynamics** (the PLDM/DINO-WM pattern; 2512.24497 found frozen encoders the *strongest* world-model backbones; arm (b) of the original plan, never run). Day-zero go/no-go costing zero training: **probe the frozen pretrained embeddings directly** on held-out layouts. **Token/path-level readout is the default** — mean-pooling a whole observation into one vector is the CLS bottleneck again; pooled features get probed too, but only as a cheap comparison point. **Candidate encoders (chosen 2026-07-13): ModernBERT-base** (149M, bidirectional, 8k ctx, MLM-pretrained with code in the mix — token states are its native product, untouched by pooling-oriented tuning) **and EmbeddingGemma-300M** (bidirectional Gemma 3 adaptation, 2k ctx — VL-JEPA's own target encoder, so corpus precedent; gated HF repo, needs a license click). Selection constraints, measured on v1 renders: full observations run 151–745 chars / 12–55 lines ≈ ~650 BPE tokens max, so 512-cap encoders (e.g. all-MiniLM-L6-v2) truncate the largest observations and are ruled out; bidirectional attention is required for context-complete per-line states — a causal LM's line states cannot see later lines of the sorted tree, polluting per-path readout (a causal-LM arm is an optional comparison, not a candidate). **Day-zero result (2026-07-13, corrected by finding 19): strong comparative result, gate-1 PASS rescinded.** `runs/frozen-modernbert-v1/probe.json`: held-out macro 100% path-keyed / 44.4% pooled vs the 28.2% *unmatched* untrained floor — but the rule requires an architecture-matched floor (+20) and the recon twin (+10), and a matched random-init encoder also scores ~100% content under path-keyed readout (finding 14's correction block), so the rule verdict is open pending step-5 re-evaluation. ModernBERT-base remains the A1 backbone; the EmbeddingGemma-300M comparison stays deferred (gated HF repo). **Gate-2 design decisions (2026-07-13):** targets render clean — decided by measurement, finding 15a. Predictor shape — the tier-2 bake-off (finding 16, corrected): slotpred beats editpred on every component in both rounds, but neither candidate passes rule 1's invalid clause under strict scoring, so slotpred is the working choice, not a rule-passed winner. **Gate 2 (finding 17, corrected): NOT passed as pre-registered.** The codebook pipeline (frozen features → label-fit linear state reader → the unchanged tier-2 checkpoint) measures near-oracle end-to-end but contains no learned dynamics over frozen features; the recipe-faithful A1 gate-2 test — a pretrained-initialized predictor in feature space, per VL-JEPA's actual recipe — has not yet run and is the next rung, judged by the hardened referee with a matched random-init-trunk control. **Ran 2026-07-14 (finding 21): the 2×2 (init × edit-weight) fails the calibration clauses on held-out layouts (no-op/invalid haze, AUC < 0.9, worse than the feature-space copy floor on state-changing); pretraining is verdict-irrelevant. A1 gate 2 not cleared. Fidelity correction: the predictor was ModernBERT's own last-3 layers (an encoder reused as predictor), NOT VL-JEPA's separate generative LLM (Llama-3.2-1B) — so this bounds that variant; a faithful generative-LLM-predictor arm is scoped and unrun.** A probe pass here only clears gate 1 — the arm then trains a predictor on the frozen features and must clear gates 2–3 like everything else.
- **A2 — temporally-masked prediction with grounded targets** (revised after review — the original "mask a subtree, predict it from visible context" is likely toothless *in this domain*: layouts are randomly placed, so hidden subtrees are nearly independent of visible context, unlike natural images whose spatial redundancy is what makes I-JEPA's masking work; and a learned target encoder can re-drift to distractor codes). The version that survives: predict the masked region of obs_t from **obs_{t−1} + action**, where the previous observation fully determines the masked content — carrying absolute state forward becomes the only way to win — with **grounded targets** (frozen-encoder or token-level embeddings, not own-encoder latents) so the target can't participate in the identifiability loophole.
- **A3 — state-equivalence invariance**: same state rendered under different banner/noise/line-order must map to the same code (line-order permutation is a particularly clean augmentation — rendering order is pure nuisance, and invariance to it forces set-structured content). Cheap term, composable with A1/A2; directly manufactures the canonical coordinates the identifiability analysis says are missing. Compatibility note for token/path-level readouts (A1's default): the invariance target is **path-keyed equivariance with order-invariant aggregation** — per-path representations follow their paths under line permutation and pooled/keyed aggregates are unchanged — not raw token-position invariance, which is ill-typed for per-token outputs. **Scoped down (2026-07-13):** gate-2 prediction targets render clean per finding 15a, which removes the target-side need for this term. **Re-opened by the adversarial review (finding 19):** the "A3 closed as unnecessary" reading of the codebook pipeline's nuisance-invariance was tautological (that pipeline's predictor never sees contamination by construction) — A3 remains live and untested for any feature-space arm, including the pending pretrained-initialized gate-2 redo.
- Crossed off by experiment: bare slot capacity (finding 11) — binding must come from the objective, not architecture.

**Track B — planning machinery, in parallel (Phase 2 of the plan, de-risked early).** Validated as a three-tier ladder that never conflates representation, dynamics, and search (review correction, adopted — "zero training" only holds on tier 1, since CEM always needs a rollout model):
1. **Oracle representation + oracle dynamics** (zero training): z = ground-truth state features, rollouts via the env's own transition semantics applied symbolically. Validates search + cost + goal exemplars + validity filter in isolation; failure here = machinery bug, full stop. **Done (2026-07-13, finding 18, corrected):** battery half via finding 15b; planner half via `plan/cem.py` — machinery validated (filtered success 73–76% vs privileged scripted ceiling 100%). The algorithm-level "CEM adds nothing" reading is re-scoped by finding 19: the unfiltered tie is real at the tested configuration, but the filtered comparison was uninformative by construction and the "+30 unreachable" claim is a hypothesis pending a committed-budget, stratified rerun.
2. **Oracle representation + learned dynamics**: train the predictor *on ground-truth features* — a diagnostic we have never had, since every predictor so far trained against representations that were themselves broken. Isolates "can this predictor architecture learn the domain's dynamics at all" from the representation question; also the natural home for the dynamics-gate evals. **Done (2026-07-13): the bake-off, finding 16 (corrected)** — slotpred dominates editpred on every component in both rounds and is the working choice; under strict clause scoring neither candidate passed rule 1 (slotpred over-fires on 13/72 invalid transitions), so "the architecture learns these dynamics" holds for state-changing/no-op behavior with a measured invalid-action miscalibration to beat.
3. **Learned representation + learned dynamics**: the real configuration — Track A's winner swaps in for the behavior gate. The current CLS encoder runs through this tier first **as an expected-negative diagnostic** (it memorizes layouts and miscalibrates no-ops), to calibrate how far a known-bad representation gets, not to validate anything.

**Deprioritized** (kept for reference, not scheduled): §7 5-seed formalization, negative-result write-up, higher-entropy banner knob, loss↔probe correlation tracking.

## Empirical findings so far

All Phase 1 numbers are **single-seed, short-run, directional** — per the plan's §7, nothing is an RQ verdict until the 5-seed paired-permutation protocol runs at a settled configuration. Artifacts: `runs/sigreg-s0/probe.json` + `probe-seen.json` (1,500-step checkpoint), `runs/sigreg-s0-4k/probe.json` + `probe-seen.json` (4,000-step), `runs/sigreg-idm-s0-4k/probe.json` + `probe-seen.json` + `idm-readout.json` (IDM arm), `runs/sigreg-v1-s0-4k/probe.json` + `probe-seen.json` (v1 data), `runs/untrained-probe.json`. All of these were produced under probe protocol **v1** and are superseded for §7 purposes (see the fidelity-audit section for measured v1→v2 shifts; `runs/untrained-probe.json` must not be reused as a baseline). Protocol-**v2** artifacts (current, artifact-scale): `runs/slot3-v1-s0-4k/probe.json` + `probe-seen.json`, `runs/sigreg-v1-s0-4k/probe-v2.json`, `runs/untrained-cls-v1-v2.json`, `runs/untrained-slot-v1-v2.json`. Caveat, narrowed: only `runs/sigreg-s0/probe-seen.json` (the 1,500-step checkpoint) predates the stratification fix — ~11% of its eval trajectories carry layouts unseen during probe fitting, so its seen-layout numbers read slightly low. The 4k sigreg and IDM `probe-seen.json` reports are stratified (they record `eval_trajs_dropped_unseen_layout`); the 1.5k report is retained as-is since the 4k stratified report supersedes it for all conclusions.

1. **The two-term recipe trains stably end-to-end** (LeWorldModel's claim replicated in the text domain): no EMA, no stop-gradient; SIGReg pulled embedding std from 0.10 to 0.78 while the Epps–Pulley statistic fell 0.42 → 0.034 and prediction loss stayed low. 1,500 steps, 4.36M params, ~45 min on Apple Silicon (MPS). *Attribution sharpened by finding 9:* stability is a joint property of the regularizer **and** the identity-at-init (zero-init AdaLN) predictor — SIGReg alone demonstrably does not prevent collapse when the predictor starts far from identity. The identity-at-init predictor is not vacuous, either: on state-changing transitions the trained CLS predictor beats the copy-z_t baseline 6.7× (`runs/sigreg-v1-s0-4k/predictor-vs-copy.json`), so the objective was genuinely engaged in all runs; on no-op transitions it is *worse* than copy (it over-applies change for invalid/no-op actions) — a miscalibration to revisit alongside the still-unrun violation-of-expectation eval.
2. **The slow-feature trap did not bite — RQ2's early answer is positive.** With per-trajectory banners active: banner-swap moves z by median 0.15 vs 1.27 for a genuine state change (ratio **0.118**; the pre-registered red line is 0.5). The banner-identity probe on the trained encoder is at chance (1.3% vs 1/64=1.6%) while the *untrained* encoder decodes banners at 5–7× chance and moves *more* on banner swaps than state changes (ratio 1.41). Training actively suppressed the distractor — notable because arXiv 2211.10831 predicted regularizer-only recipes fail here.
3. **State is encoded, but layout-entangled — the central Phase 1 problem.** Per the stratified, audited report `runs/sigreg-s0-4k/probe-seen.json` (probe heads fit and evaluated on disjoint trajectories over shared layouts): linear probes decode content-class-given-existence at 46.3% (floor 17.5%) and cwd at 18.0% (floor 9.8%) on seen layouts — macro 40.3% — while *held-out* layouts stay at chance (macro 25.3–25.7% vs ~26% floor; untrained encoder identical). The JEPA objective forces real state into z, but not as a path-consistent code that transfers to new arrangements. (The 1.5k-checkpoint report `runs/sigreg-s0/probe-seen.json` predates stratification and is superseded; see caveat above.)
4. **Training scale is not the lever.** 4,000 steps vs 1,500 (2.7×, same seed/data/arm): held-out macro unchanged (25.3% vs 25.7%), seen-layout macro unchanged (38.5% vs 39.3% linear, 48.7% vs 49.1% MLP) — both plateaued by 1,500 steps. Banner invariance held at both scales (audit ratio 0.11–0.12).
5. **Large linear-vs-MLP gap on seen layouts** (linear 39%, MLP 49%; existence balanced-accuracy 57% vs 69%): more state information is present in z than is linearly organized — but on held-out layouts even the MLP is at chance, so the extra information is layout-entangled too.
6. **Measured chance floors** (val split; now embedded in every probe report): cwd majority-class 9.8%, content majority-class 17.5%, existence balanced-accuracy 0.5, banner 1.6%. IDM heads: verb majority 25.8%, arg1 majority 2.3% (299 distinct), arg2 blank-prior 62%.
7. **IDM does not fix layout transfer — but it localizes the failure.** The `sigreg+idm` arm (`runs/sigreg-idm-s0-4k/probe*.json`): held-out-layout probes unchanged at chance (macro 25.9% linear); stratified seen-layout probes essentially equal to sigreg-only (40.0% vs 40.3% linear; 54.0% vs 52.0% MLP). The per-head post-hoc readout is the informative part (saved artifact: `runs/sigreg-idm-s0-4k/idm-readout.json`, self-auditing with floors computed from the same sampled labels): on **held-out layouts**, the IDM head decodes verb at 65.0% (artifact floor 27.1%) and **arg1 — which path was touched — at 21.5%, ~8.8× its 2.45% floor**, while arg2 (62.1%) sits barely above its blank-prior floor (60.5%) — i.e., the arg2 head learned essentially nothing beyond the prior. So *change* information is partially layout-general — the encoder can often tell which path an action touched even on unseen arrangements — but that delta-level code never composes into linearly-decodable absolute state. IDM also reshapes the geometry (z_std 1.89 vs 0.77, prediction loss 6× higher) without changing the transfer outcome; banner invariance holds (ratio 0.113).
8. **Stratified seen-layout reports** (audit caveat closed): dropping the ~11% of eval trajectories with fit-unseen layouts raised sigreg-4k seen-layout macro from 38.5% to 40.3% linear (52.0% MLP), as predicted. By-depth breakdown: existence decoding degrades with path depth (0.63 → 0.58 → 0.54 balanced accuracy at depths 1/2/3, linear), consistent with weak path compositionality; content-given-existence unexpectedly *rises* with depth (0.36 → 0.41 → 0.52) — unexplained, flagged rather than interpreted.
9. **First observed SIGReg-insufficient collapse — root-caused to predictor initialization.** The initial slot-encoder run collapsed outright (z_std 0.10 → 0.0007, prediction loss 8×10⁻⁵, SIGReg statistic stuck elevated) — the first genuine JEPA collapse of the project, despite SIGReg. Root cause: the CLS `Predictor` is the *identity at initialization* (zero-init AdaLN gates), so prediction loss starts near zero and there is no incentive to collapse; `SlotPredictor`'s self-attention residuals were not zero-initialized, prediction loss started at ~1.5, and crushing the encoder to a point was the fastest descent path — the regularizer lost the race. Two fixes, both verified by smoke run (z_std rising and stable): zero-init gates on the attention residuals (restoring identity-at-init — LeWorldModel's zero-init principle applied uniformly), and slot-structured regularization (per-slot SIGReg over 16× the samples in 64-d, plus joint variance/decorrelation against inter-slot duplication). Lesson recorded: *identity-at-init predictors are load-bearing for collapse-free two-term JEPA training* — arguably a sharper statement of why zero-init AdaLN matters than the corpus papers make explicit.
10. **Layout diversity doesn't fix transfer either — and it unmasks the seen-layout signal as memorization.** `sigreg` on v1 (1,566 train layouts, ~2 trajectories each vs v0's 323 × ~6; `runs/sigreg-v1-s0-4k/probe*.json`): held-out layouts still at chance (linear macro 24.5%). Meanwhile seen-layout decoding *fell* (linear macro 40.3% → 33.9%; MLP 52.0% → 41.6%) — with fewer trajectories per layout there is less layout-instance memorization available, and no compositional code arose to replace it. The seen/held-out gap narrowed from the wrong side. Banner invariance strengthened further (audit ratio 0.075). Taken together with findings 4 and 7: training scale, inverse dynamics, and data diversity are all exhausted as levers; the architecture hypothesis (single-CLS-vector bottleneck) is now the leading explanation, per queue item 4. The v1 `sigreg+idm` rerun was skipped — IDM's transfer effect was null on v0 and nothing in the v1 result changes that calculus.

11. **Slot latents do not fix held-out transfer.** `runs/slot3-v1-s0-4k` (audited recipe, protocol-v2 battery, artifact scale): held-out linear macro 28.0% vs its architecture-matched untrained floor 28.4% — no margin at all; seen-layout macro 37.3% linear / 44.9% MLP, no better than CLS. Caveat: the run is budget-bound, not converged (z_std 0.49 and still rising toward the predicted ~0.96 equilibrium at step 4,000), and its final 1,000 steps used a degraded resume (fresh optimizer moments) after connectivity-related kills.
12. **The slow-feature trap is recipe-dependent — and the slot recipe resurrects it.** The trained slot encoder decodes banner identity at **29.3% (19× chance)** and its banner-swap ratio is **3.22** — banner swaps move z three times more than genuine state changes (red line: 0.5; the CLS arm sits at 0.075). Training *amplified* banner sensitivity beyond the untrained slot encoder (22.8%, ratio 1.19). Mechanistic reading: the slot recipe's much stronger variance demand (25:1 hinge + per-index SIGReg over 16 slots) needs cross-sample variance, and the banner is the cheapest source — static within trajectory (zero prediction cost), varying across trajectories (free variance). This reframes finding 2: SIGReg didn't defeat the 2211.10831 trap — the CLS recipe's *weak* variance pressure did; crank the variance demand and the trap bites exactly as the paper predicts. Both regimes now observed in one domain with instrumentation.
13. **Trained-below-untrained confirmed at artifact scale, both architectures.** Under protocol v2 on held-out layouts: CLS-trained 26.0% vs CLS-untrained 28.2%; slot-trained 28.0% vs slot-untrained 28.4% (linear macro). JEPA training in this domain consistently *removes* linearly-decodable held-out state relative to random features — the strongest formulation of the layout-entanglement result, now at full scale. Meanwhile the protocol reviewer's banner-probe worry resolved benignly for the CLS arm: at artifact scale under v2, CLS-trained banner accuracy is 1.26% ≈ chance (the small-scale 4.6% was cluster noise, as they cautioned), so finding 2's CLS claims stand with finding 12's recipe-dependence caveat.

14. **A1 day-zero probe: frozen pretrained text features dissolve the representation problem — zero training (2026-07-13).** `runs/frozen-modernbert-v1/probe.json` (ModernBERT-base 149M frozen, data v1, protocol-v2 head fitting, 20k fit / 5k eval states, regime `both`; tool: `probes/frozen_probe.py`): under the **path-keyed readout** (per-line mean-pooled token states keyed by the path each line describes), held-out-layout probes are **perfect** — cwd 100% linear (floor 10.3%), content-given-existence 100% linear at every depth (floor 17.5%), existence structural 1.0 (the line keying *is* the parser; verified exact over all 25k states, 0 mismatches). Even the **pooled** readout (mean over all tokens — deliberately the CLS bottleneck) scores **44.4% linear macro on held-out layouts vs untrained-CLS 28.2% and trained-CLS 26.0%**: a frozen pretrained encoder beats every from-scratch arm by ~16 points under the readout all of Phase 1 used. Honest caveats: under path-keyed readout, content/cwd are near-trivial reads of in-line tokens — the substance is that pretraining keeps them *linearly available under contextualization*, which our JEPA-trained encoders demonstrably destroyed (finding 13); and nuisance is faithfully preserved too — pooled banner-swap ratio 1.17 (red line 0.5), banner identity decodable at 45% pooled / 31% from path-line means, and a banner swap moves a path line's vector 0.75× as much as a genuine state change moves *other, textually unchanged* lines. 24. **Setup/data-vs-JEPA adversarial review — the wall is held-out-layout *binding over a non-canonical representation*, not sparsity or the distractor; and clean full-obs is genuinely off-regime for JEPA (2026-07-15).** [Full record: `reviews/2026-07-15-setup-data-vs-jepa-review.json`; 29 findings, 37 agents; tested the hypotheses that our discrete/sparse synthetic data and our banner distractor cause the negatives.] **The decisive evidence is a dissociation on our own artifacts:** the *identical* sparse dynamics (mean 1.078 of 301 slots change per action, 92% exactly 1 slot, 27.5% no-ops with frozen-feature delta exactly 0.0) learned over the **oracle symbolic** representation clear the gate and transfer to held-out layouts (tier-2 slotpred 0.277 vs copy 1.076, AUC 0.906), but over **frozen ModernBERT features** collapse to copy on held-out (genllm 63.95 vs 63.13, AUC 0.803) *while beating copy on seen layouts* — sparsity is identical in both, only the representation differs, only one fails. **Hypothesis verdicts:** (a) *sparse/discrete data* — a real modality mismatch and a contributing amplifier (copy is the identity-at-init default and is exactly optimal on 27.5% of transitions; V-JEPA's masked targets are dense and structurally un-copyable), but **NOT the root cause** (the dissociation), and reducing the no-op rate / up-weighting changed slots will not fix transfer (edit-weight 10 already failed by hazing copy slots). (b) *distractor ill-conceived* — **no**, the per-trajectory banner is a correct slow-feature analog that bites the slot recipe at 19× chance, but it is a **red herring for these negatives**: the gate-2 arms render targets clean and have no variance regularizer, so it cannot drive the collapse; keep it only as an A3/RQ2 instrument. (c) *wrong regime for JEPA* — **partly, in a specific fixable way**: clean full-obs is a lossless deterministic serialization of the exact state (`[cK]` content tokens; the parser is a tested inverse), precisely the regime JEPA.md §3.2 reserves no edge for ("reconstruction works well for text"), so JEPA's differentiating mechanism (a target encoder that discards nuisance) is **definitionally inert** — which is why the recon twin ties/beats JEPA (finding 22) and random-init ties pretrained. *Not* a blanket disqualification: the symbolic predictor transfers ("predict in representation space" works), the action→state map is high-MI deterministic, and the binding wall is modality-independent (recon shares it). **Load-bearing divergences, ranked:** (1) held-out-layout compositional binding over the frozen contextual interface — the real wall, modality-independent; (2) clean serialization = zero irreducible detail to abstract — off-regime; (3) frozen off-the-shelf ModernBERT is *non-canonical* — bidirectional contextualization smears a 1-slot edit across ~15 layout-idiosyncratic feature slots (~55% of the target-delta is contextual haze), the concrete mechanism that fails the frozen-feature arm where the symbolic arm succeeds — **kill: do not regress raw frozen contextual features; predict in a canonical/decoded space**; (4) sparse near-copy target + identity-at-init predictor = copy attractor (amplifier); (5) discreteness → continuity-exploiting planners inapplicable (consecutive latent velocity cos ≈ 0) and the factored ~837k-arg action space starves CEM (planner-only, downstream of the representation wall). **Keep:** the symbolic env as ground-truth; the tier-2 symbolic predictor (it clears the gate *and* transfers — the working world model, and the honest product path is frozen features → symbol grounding → symbolic dynamics, i.e. the finding-17 codebook route); the held-out split + 5-seed hardened battery. **Kill:** regressing raw frozen ModernBERT features as the JEPA target; clean full-obs as the *primary* regime (demote to the parsing smoke test it was always meant to be, terminal-jepa.md §4); banner-strengthening and any negative attributed to it; the CEM +30-over-random bar in the full-obs combinatorial action space; further no-op-rate / edit-weight tuning (provably does not fix transfer).

## Data redesign directions (active decision, 2026-07-15)

The review's throughline: **move the observation channel into JEPA's regime while keeping the symbolic oracle for evaluation, and use the reconstruction twin as the discriminator** — because the load-bearing wall (rank-1 held-out binding) is the one thing a modality change might *not* fix; if transfer still fails after moving fully into JEPA's regime, that isolates compositional binding as a modality-independent wall (a decisive result). Ranked candidates:

1. **Verbose + partial-obs *text* (primary, ~1 week, reuses the eval stack).** Render realistic lossy command output — `ls -la` with timestamps/permissions/sizes and *OS (unsorted) ordering*; `cat` returning pseudo-random byte bodies whose *content class* is a needle (no `[cK]` token); interleaved PIDs/hostnames *through* the state lines — and switch to `render_partial` (last command's stdout only) with a W>1 / recurrent predictor. Injects the irreducible in-band nuisance JEPA's abstraction is *for* (attacks divergence 2), kills the path-keyed-parser and `[cK]` shortcuts that make the env a toy, and restores dense action-coupling (consecutive-obs overlap 97.5%→0.3%, dissolving the copy attractor, divergence 4). Signal it worked: the recon twin's parse-to-ceiling edge collapses AND a JEPA/EMA arm holds; random-init no longer ties pretrained.
2. **Terminal-as-pixels → literal V-JEPA (definitive, higher effort).** Render screenshots (monospace raster) and run the actual V-JEPA-2-AC recipe (EMA ViT target encoder, spatiotemporal multi-block masking, action-conditioned predictor). Exactly what a computer-use agent perceives and the exact recipe the corpus validated; masked targets are structurally un-copyable (divergence 4) and the EMA target replaces non-canonical frozen ModernBERT (divergence 3). Largest lift: a real V-JEPA training stack we don't have.
3. **Correlated/cascading symbolic dynamics (cheapest, stays where the gate is already cleared).** Macro-actions touching several correlated slots; rule-based content (`content = f(path)` or a per-layout "project type") so masked-content prediction has real MI (currently I(content;visible) ≈ 0.01 bits — masking is provably ill-posed); drop the no-op mass from the loss. Attacks the binding wall while keeping the symbolic target that already transfers; does *not* test JEPA's abstraction edge.
4. **Real recorded agent sessions (maximal realism, weeks).** Record real/LLM-driven shell sessions against a sandbox; snapshot FS state for ground-truth. The target distribution itself; best pursued *after* a cheaper redesign confirms the regime hypothesis.

**Pragmatic note:** the symbolic route (decode → symbolic dynamics → plan) already clears the gate, so a *working* terminal agent does not require cracking the frozen-feature-dynamics wall — that is the product path; directions 1–2 are the science (does JEPA earn its keep in-regime).

## Implementation-fidelity audit (2026-07-10)

After the collapse incident (finding 9) exposed an undocumented deviation from the papers' recipes, three parallel auditors compared every component against freshly-fetched source papers (LeJEPA, LeWorldModel, VICReg, DiT, Causal-JEPA, V-JEPA, 2512.24497, 2211.10831), running numerical checks where claims were quantitative. Full reports are preserved in the session tasks; summary:

**Confirmed deviations, fixed (one changeset, 2026-07-10):**
- `vicreg_var_cov` divided the covariance penalty by an extra √d with no paper basis — a 32× weakening at d=1024 of the *only* inter-slot decorrelation force. Restored to VICReg's exact normalization.
- Pooled per-slot SIGReg was 87%-satisfied by static slot-identity offsets (measured on the live checkpoint) and is provably blind to inter-slot duplication (16 identical slots score at the noise floor). Replaced with `sigreg_per_index` — a separate Epps–Pulley test per slot index (16×256 = 4,096 univariate tests), under which static offsets are themselves penalized; regression test added.
- The 3-step rollout term flowed gradients through its target — an extrapolation existing in no source (LeWorldModel trains 1-step only; 2512.24497 rolls out against frozen encoders with truncated BPTT) that rewards latent contraction, and the audits' consensus explanation for the z_std ≈ 0.77 plateau (our SIGReg scale convention leaves only a tiny restoring force near 1.0). Rollout target now detached by default; `--rollout-full-bptt` preserves the old behavior for ablation.
- The SlotEncoder applied an affine LayerNorm before its output head — the exact pattern LeWorldModel explicitly engineered out (final LN "prevents the anti-collapse objective from being optimized effectively"). Removed; legacy pre-fix checkpoints load with a loud warning.
- SIGReg sketch 512 → 1024 directions (paper value); weight decay now excludes 1-D params/embeddings/queries (decay was silently pulling zero-init gates closed at ~1.2%/4k steps).
- Probe protocol v2: features standardized on fit-split statistics before head fitting (arms differ up to 5× in embedding scale), lr 1e-2 → 3e-3 (measured better at the fixed budget), seeded head init, report provenance read from actual defaults, legacy-checkpoint loads recorded in the report and gated behind `--allow-legacy`. Cross-protocol slack (corrected by adversarial review, which measured it): **≤1 pt macro for trained arms, but up to ~3 pt macro / ~6 pt per-metric for untrained (low-scale) features** — v2 removes an 8–20× effective-lr handicap that the old protocol imposed on untrained baselines, raising the sanity floor ~3 pts. Consequence: archived probe.json files are incomparable at the metric level; checkpoints get re-probed under v2 rather than slack-adjusted, and §7 now pins architecture-matched, same-protocol untrained baselines (amendment recorded in the plan).
- Truncation counter in the data loader (observations that exceed 640 tokens would silently drop state entries while keeping the distractor header; currently 0 occurrences, now impossible to miss).

**Measured and cleared:**
- Our EP discretization is unbiased (argmin exactly σ=1.0) — the z_std plateau is not a quadrature artifact.
- Probe-budget fairness for the slot-vs-CLS comparison: measured in strong-signal, weak-signal (near-chance), and real-checkpoint regimes — differential ≤1 pt everywhere, and the fixed budget slightly *favors* the wider head. A false "slot stays at chance" verdict cannot come from probe underfitting; any verdict within ~2 pts of floor is protocol-sensitive and triggers a per-arm lr sweep.
- Banner-swap audit is truly state-isolated (noise-line confound tested: nil). No train/val leakage (splits disjoint, vocab closed, 0 UNK).
- No further identity-at-init violations anywhere gradients flow (both predictors float-exact identity).

**Known deviations, documented and kept:**
- Scale conventions: prediction loss is mean-over-dims and our SIGReg statistic is N-free (papers: sum-over-dims, ×N). At d=256 the two deviations approximately cancel — *by coincidence* — which is why CLS runs matched paper behavior; recorded as the project's convention rather than retrofitted, since λ was tuned under it.
- Batch gives 128 embeddings/step to SIGReg — exactly LeJEPA's validated floor.
- `noise_seed` = within-split trajectory index, so val replays train's noise-line streams: benign for state metrics (noise is state-independent), but robustness to *novel* dynamic noise is untested by construction.
- RQ2 caveat (weakens finding 2): our banner is a far weaker trap than 2211.10831's full-frame distractor (one line, ~3% of tokens, 6 bits of entropy vs image-scale noise), and their failing recipes were VICReg/SimCLR-based, not SIGReg+identity-init. The untrained encoder does latch onto the banner (5–7× chance), so the trap direction exists — but "positive surprise vs the literature" overstated it; a higher-entropy multi-line banner knob is queued before any RQ2 write-up.

## Operational learnings (matter for reproducing on another machine)

- **Python 3.14**: `random.Random` no longer accepts tuple seeds — all RNG streams are string-keyed (`f"traj:{seed}:{split}:{i}"`); str seeding is sha512-based and stable across processes.
- **macOS jetsam killed long training runs** under system memory pressure. Fixes: tokenized data stored as int16 tensors (~265MB peak RSS for the full split, was ~1GB+ of Python lists), and training drops FsState objects it doesn't need.
- **MPS produced NaN under memory pressure** — the identical run (same seed, same batches) replayed clean, so it was environmental, not a math bug. The trainer now skips any step with nonfinite loss or gradient norm (weights can't be poisoned), clips gradients at norm 1.0, and logs a `skipped` counter in every `train.log` line. In practice the condition fires steadily at ~2% of steps on this machine (82 skipped in the 4,000-step run) and training proceeds unharmed. Caveat: clipping is a mild dynamics change vs the first 1,500-step baseline; all §7-protocol arms will share the guarded config.
- **Dynamic padding** (trim batches to their longest observation, ~250 of 640 tokens typical) roughly halves step time.
- **Ported venvs keep stale shebangs**: after the repo (including `.venv`) was copied to a new path (2026-07-10), console scripts like `.venv/bin/pip` still point at the old absolute interpreter path and fail with "bad interpreter"; `.venv/bin/python` is a symlink and works — invoke tools as `.venv/bin/python -m pip ...` (etc.).
- **A1 frozen-probe dependency**: `transformers` (5.13 tested) added to the venv; ModernBERT-base (~600MB) downloads from HF unauthenticated on first use. Full probe run (25k observations, MPS, fp32): ~13 min wall clock.
- **This machine cannot sustain the workload (2026-07-13/14)**: three background training runs killed in one day (editpred twice, round-4 once), and concurrent multi-agent review + training forced a hard reboot. Rules adopted: strictly one heavy process at a time, reviews and training never concurrent, checkpoints every 200 steps — and development now continues on a bigger machine (see Handoff).
- Every training run streams JSONL to `runs/<name>/train.log` (step, loss, pred, reg, z_std, skipped, sec) — `tail -f`-able.
- Probe reports are self-auditing: `probe.json` embeds the measured chance floors, fit/eval state counts, the eval split (`val-heldout-layouts` vs `seen-layouts-heldout-trajectories` via `--seen-layouts`), the count of eval trajectories dropped for carrying fit-unseen layouts (stratification), and probe-fit hyperparameters.
- IDM training logs per-head losses and accuracies (`idm_verb_*`, `idm_arg1_*`, `idm_arg2_*`) — arg1 (which path was touched) is the load-bearing head for the path-code hypothesis. The completed IDM run predates this logging; its per-head readout is saved in `runs/sigreg-idm-s0-4k/idm-readout.json` (regenerable via `probes.idm_readout`).
- Phase 1 has CPU smoke tests (`tests/test_phase1.py`: batch → forward → every loss arm backward → probe pipeline, over a tiny generated dataset). Run the suite with `.venv/bin/python -m unittest discover -s tests` to include them (20 tests); the system python runs Phase 0 only and skips them (stdlib-only property preserved).

## Design decisions a reviewer should know (with rationale)

- **Runtime state is not capped** at the layout bounds (≤12 dirs / ≤20 files apply to generation only; vocabulary bounds runtime at 42/258): a hard cap would force the goal-exemplar constructor to delete entries, breaking minimal-edit semantics. Realized max is reported in `summary.json`.
- **The invalid-action quota is enforced per step at the trajectory level** (policy choice overridden with probability q), so the realized invalid rate tracks the quota (15.1% vs 15% in v0) regardless of policy mix.
- **Layout identity excludes cwd** (tree + contents only) — cwd is initial state, not layout; including it would let the same arrangement appear in both splits.
- **The full-obs parser is one shared component** used by four consumers (classical baseline, planner validity filter, goal-exemplar construction, and Phase 3's belief tracker) — planners stay nonprivileged because everything they use derives from the observation.
- **Phase 2 is machinery validation, not superiority**: in full-obs, parser+BFS is a nonprivileged ceiling that will likely solve the domain outright; superiority claims are reserved for regimes where parsing fails (Phase 3).
- Full review history (6 rounds: architecture → evaluation fairness → measurement precision) is baked into terminal-jepa.md §3–§7; every threshold in §7 was fixed before experiments began.

## Known risks / open questions

- The compositional generalization gap (finding 3) may not yield to training scale — if the 4k run and IDM arm don't move held-out probes, the honest possibilities include: the CLS-vector bottleneck (a single 256-d vector for a 300-line observation; token-level latents are the escape hatch, noted in the plan §5), or insufficient layout diversity at the dataset level.
- Probe heads are trained with fixed budgets (400 steps, lr 1e-2); underfitting the probes would depress all arms equally but absolute numbers should be read with that in mind.
- The `recon` twin and `vicreg`/`sigreg+tempsim` arms are implemented but not yet run.
