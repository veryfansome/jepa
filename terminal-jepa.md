# Terminal-JEPA: a JEPA world model for a sandboxed terminal agent

Project plan. Background and citations live in [JEPA.md](JEPA.md); this document only covers what we're building and how we'll know if it worked. Implementation status and findings to date: [terminal-jepa-status.md](terminal-jepa-status.md).

## 1. Hypothesis

The JEPA world-model recipe — encode observations into latents, train an action-conditioned predictor in latent space with an anti-collapse regularizer, plan by minimizing latent distance to an encoded goal — transfers from pixels/robot-states to **text observations and discrete shell commands**. If it does, we get two things no pure-LLM agent has: (a) a learned state representation that tracks the *true* sandbox state rather than surface text, and (b) millisecond-scale counterfactual evaluation of candidate command sequences before executing them (including destructive ones).

Nothing in the JEPA corpus demonstrates this — every planner in those papers samples low-dimensional continuous actions, and PLDM explicitly assumes full observability. This is research, not application; a clean negative result is a valid outcome.

## 2. Research questions

- **RQ1 (representation):** Do JEPA latents trained on terminal trajectories linearly decode the true environment state (cwd, file existence, file contents) — and do they beat a compute-matched text-reconstruction baseline?
- **RQ2 (robustness):** Where do the known JEPA failure modes bite in this domain — specifically the slow-feature trap (per-trajectory-static banners satisfying the objective) and compounding rollout error?
- **RQ3 (planning):** Does latent-space planning over command sequences reach goal states more reliably / in fewer environment steps than random search — and how far does it sit below the nonprivileged parser+search ceiling? (Planner A vs the greedy LLM is reported descriptively but is not a pass bar; LLM comparisons belong to RQ4.)
- **RQ4 (hybrid):** Does "LLM proposes, JEPA world model ranks" beat either component alone? (This is the position paper's Mode-1/Mode-2 split instantiated for text.)

## 3. Phase 0 — the sandbox environment

A synthetic, in-process filesystem world. No real shell, no containers — determinism and ground-truth access matter more than realism at this stage.

- **State:** a small tree (≤3 levels), file contents drawn from a small vocabulary, plus cwd. The ≤12-directory / ≤20-file caps apply to *layout generation only*: runtime states may grow past them through mkdir/touch/write and are bounded by the vocabulary (42 possible dirs, 258 possible files); datagen reports the realized maximum. Runtime caps are deliberately not invariants — a capped state would force the goal-exemplar constructor to *delete* entries to satisfy a FileExists predicate, breaking minimal-edit semantics (§5).
- **Actions — factored, enumerable:** `(verb, arg1, arg2)` with `verb ∈ {cd, ls, cat, mkdir, touch, rm, cp, mv, write}` and args drawn from the path/content vocabulary. Factoring sidesteps the unbounded-command-space problem for now; free-form command strings are Phase 3.
- **Observation:** rendered text, two regimes:
  - **Full-obs (start here):** serialized tree + cwd after each action — matches PLDM's full-observability assumption, isolates the representation question from the memory question.
  - **Partial-obs (hard mode):** only the prompt + stdout of the last command (`ls` shows one directory, `cat` one file). This is the real terminal condition.
- **Distractor knobs (for RQ2):** injectable observation noise — (i) a banner string drawn *per trajectory* and held fixed within it; this, not a globally constant banner, is the slow-feature trap from arXiv 2211.10831 — the distractor must vary across trajectories to let the encoder satisfy variance regularizers by copying it, while staying trivially predictable in time (a globally constant banner would simply be ignored, since constant features are what those regularizers penalize); (ii) *dynamic* noise resampled every step (timestamps, fake PIDs); (iii) none. Banner identities are sampled i.i.d. from one fixed vocabulary shared across train/val splits, independent of layout, policy, and split (the same banner recurs across layouts) — otherwise the §4 audit confounds banner decoding with layout/trajectory decoding, and split-disjoint banner vocabularies would break the identity probe outright.
- **Ground truth API:** the env exposes true state features for probing — cwd, file-existence bitmap, per-file content class, plus each trajectory's banner identity (so the slow-feature audit has a real label). All label spaces are defined over a single **shared path/content enumeration** used by every layout, so probes stay well-defined across layouts (see the split definition below).

**Data generation:** trajectories from a mixture of policies (typed-random actions; scripted goal-reachers with ε-noise), mirroring PLDM's reward-free suboptimal-offline-data setting. Typed-random sampling draws from the *valid* action set (preconditions satisfied: `cd` targets an existing directory, `mkdir` a fresh path with existing parent, etc.); an explicit invalid-action quota (default 15%) is enforced **per step at the trajectory level** — with probability q the policy's chosen action is overridden by a sampled invalid one, so the realized invalid rate tracks q regardless of policy mix. Naive uniform sampling over the factored space would be dominated by failed commands and mostly teach "state doesn't change," but error/no-op semantics are real dynamics the model must learn, hence a quota rather than zero. Every transition is labeled by type (**state-changing / valid-no-op** (e.g. `ls`) **/ invalid**) and all Phase 1–2 metrics are reported split by type. Datagen additionally reports invalid-action coverage broken down by failure type (nonexistent path, wrong argument type, precondition violation) — planner exploitation of undertrained invalid actions is a known risk, measured in §5. Target ~100k–1M transitions; generation is pure Python, fast, and can be regenerated freely as the env evolves.

**Layout split, precisely:** all layouts draw paths and contents from the one shared enumeration; a *held-out layout* is an unseen arrangement (tree shape + content assignment) of those known symbols. Layout identity for dedupe and split membership is keyed on tree shape + content assignment **only** — cwd is initial *state*, not layout, and is excluded from the key so the same arrangement can never appear in both splits under different cwds. This is compositional recombination, not new-symbol generalization — a weaker but well-defined claim (PLDM's pixel-maze layouts never face this distinction; symbolic domains do). New-symbol layouts are a stretch goal gated on the compositional action encoder in §4.

**Exit criteria:** env round-trips (action → state → rendering) with property-based tests; dataset generator produces train/val splits with held-out *goal predicates* and held-out *tree layouts* as defined above, and reports the transition-type mix.

## 4. Phase 1 — world model + probing

Recipe follows LeWorldModel (arXiv 2603.19312) as closely as the modality allows: two-term loss, no EMA, no stop-gradient, end-to-end.

- **Observation encoder:** small transformer over tokenized observation text → single latent (CLS-style), z ∈ R^256. Two arms: (a) trained from scratch (~5M params), (b) frozen small pretrained text encoder + trainable projector — the text analogue of frozen-DINOv2-plus-projector, which arXiv 2512.24497 found to be the strongest world-model backbone in vision.
- **Action encoder:** compositional — (verb, arg1, arg2) rendered as a short token sequence through a small encoder, *not* a per-tuple embedding lookup. A tuple lookup can't represent unseen combinations (breaking held-out-layout evaluation) and permanently blocks new-symbol layouts; token-level encoding matches how the observation encoder already handles paths.
- **Predictor:** small causal transformer (~5–10M params), action conditioning via zero-init AdaLN (best-performing conditioning in 2512.24497 and used by LeWorldModel), context window W=1 — Phase 1 trains and evaluates in full-obs + nuisance regimes only; partial-obs configurations (longer W, memory) live in Phase 3.
- **Loss:** L2 teacher-forced latent prediction + multi-step rollout loss (k ∈ {1, 3}; rollout training is the load-bearing anti-compounding measure per 2512.24497) + λ·SIGReg (λ=0.1 default). The two-term default follows LeWorldModel, but three ablation arms are mandatory, not optional: VICReg in place of SIGReg (EB-JEPA's head-to-head), an **inverse-dynamics (IDM) auxiliary loss**, and a temporal-similarity loss. EB-JEPA and PLDM both report IDM as load-bearing for action-conditioned world models (removing it collapses EB-JEPA's Two-Rooms planning success from 97% to 1%), and 2211.10831 found inverse dynamics robust to exactly the distractor failure RQ2 tests — so if the minimal recipe collapses or latches onto banners, the IDM arm distinguishes "terminal-JEPA hypothesis fails" from "missing known control."
- **Baseline (for RQ1):** a compute-matched generative model — same encoder trunk, decoder head trained to reconstruct the next observation *text*. Regime matters: in clean full-obs the observation *is* a machine-readable serialization of the probe targets, so the reconstruction twin (or a hand-written parser) sits near the probing ceiling by construction — parity there is expected and fine. The abstraction premise is only tested where observations contain nuisance detail to abstract away; see the regime-scoped pass criteria below.
- **Scale:** ≤20M params total. Runs on a single consumer GPU or Apple Silicon (MPS); EB-JEPA (github.com/facebookresearch/eb_jepa) is the reference implementation to crib training-loop structure from, though our modality means custom encoders regardless.

**Evaluation / exit criteria:**

1. **Probing (RQ1), regime-scoped:** linear + MLP probes from z to ground-truth state features, on held-out layouts — the linear probe is the pass bar (RQ1 asks whether state is *linearly* decodable); the MLP probe is reported for the linear-vs-MLP gap, which diagnoses information that is present but not linearly organized. *Clean full-obs is a smoke test only* — it is effectively a parsing benchmark, and parity with the reconstruction twin is acceptable there. Pass/fail is decided in the nuisance regimes (per-trajectory banners + dynamic noise): there, JEPA probes must beat (a) an untrained encoder and (b) the reconstruction twin by the pre-registered margins in §7.
2. **Slow-feature audit (RQ2):** with per-trajectory banners active, two measurements: (a) probe z for banner *identity* (a real label now that banners vary across trajectories) vs state content — the 2211.10831 failure mode is banner decodable, state not; (b) eval-time banner swaps — re-render the same state under a different banner and compare ‖Δz‖ against the ‖Δz‖ of a genuine state change (a probe-free sensitivity readout). Run across the SIGReg / VICReg / +IDM arms; 2211.10831 predicts the regularizer-only recipes fail and the IDM arm resists.
3. **Rollout drift (RQ2):** k-step prediction error vs k, with and without rollout-loss training.
4. **Violation-of-expectation:** prediction error should spike on physically-impossible transitions (file appears without a creating command) — LeWorldModel's sanity check that the model learned dynamics, not statistics.

## 5. Phase 2 — planning (full-obs only)

Phase 2 runs entirely in the full-obs regime; partial-obs planning is deferred to Phase 3, where goal specification has to change (see §6). Framing matters: in full-obs a trivial parser + classical search is a *nonprivileged* ceiling (see Baselines), so Phase 2 cannot claim superiority over classical planning — its claim is **machinery validation**: the learned latent cost + search apparatus works end-to-end. Superiority claims are reserved for regimes where parsing fails (partial obs, free-form real-shell text — Phase 3).

- **Goal specification — predicates, not states:** a goal is a predicate over state (e.g., "file `b/notes.txt` exists with content class 3"), which many states satisfy, and success is predicate satisfaction checked against ground truth — the cost must match that. Encoding one full satisfying state as z_g (the naive goal-image analogue) penalizes irrelevant differences, and even min-distance over a *sampled* set of satisfying states leaks irrelevant state whenever the sample misses the plan-time configuration (current cwd, incidentally created files) — the planner then chases the nearest exemplar rather than the predicate. So goal exemplars are constructed **from the plan-time observation**: parse the current observation into a symbolic state — with the same parser used by the validity filter and the classical baseline, keeping planners nonprivileged — then apply the minimal predicate-satisfying edit (plus a few variants) **through the env's validated state constructor**, so irrelevant features match by construction and residual latent distance is predicate-relevant; exemplars are recomputed at every replan. The validated constructor matters: a naive minimal edit can produce an invariant-violating state (a file under a nonexistent parent directory) that the encoder has never seen and can never see — its embedding, and any latent distance to it, is undefined behavior. Property-based tests per predicate type assert every generated exemplar satisfies both the env invariants and the predicate. That parser is now load-bearing in three places (baseline, validity filter, goal construction): one shared, tested component, not three ad-hoc ones. Known residual: plans with unavoidable side effects (an intermediate `cd`) still incur small irrelevant penalties — documented, not solved. (Predicate-masked distance would be cleaner but requires token-level latents; the single-vector z rules it out — revisit if we move to per-token latents.)
- **Planner A — search over factored actions:** CEM / beam search over the enumerable action space, horizon H ∈ {3..8}, cost = ‖ẑ_H − z_g‖₂, receding-horizon (replan each step). Sampling-based per 2512.24497's finding that CEM beats gradient planning on non-smooth landscapes — and a discrete action space is maximally non-smooth. Run with and without a **candidate-validity filter** that parses the current observation and drops precondition-violating actions — nonprivileged in full-obs (the parser consumes exactly what the planner sees, analogous to a real agent checking a path before acting); the with/without gap measures how much CEM exploits undertrained invalid actions.
- **Planner B — LLM-propose / JEPA-rank (RQ4):** an off-the-shelf LLM proposes K candidate action sequences from the observation + goal text; the world model rolls each out in latent space and executes the argmin. No LLM fine-tuning. **Lowering spec:** proposals are constrained to the factored grammar (structured output: a list of (verb, arg1, arg2) tuples) and passed through a deterministic parser/canonicalizer; unparseable or out-of-vocabulary proposals are dropped, and **proposal-validity rate is reported as its own metric**, separate from ranking quality — otherwise RQ4 measures invalid-output handling rather than whether JEPA ranking helps.
- **Baselines:** random shooting; greedy LLM (execute its top valid proposal unranked); **parser + BFS/A\*** — parse the full-obs observation into a symbolic state and search classically. The parser+search baseline is *nonprivileged* (it consumes exactly the planner's input) and will likely solve this domain outright — that is the honest ceiling. BFS on true state (privileged) is kept only as a sanity check on the parser.
- **Metrics:** goal-reach (predicate-satisfaction) success rate on held-out predicates AND held-out layouts, env steps consumed, wall-clock per planning call; for planner B — proposal-validity rate, plus how often JEPA ranking changes the LLM's choice and whether the change helps (the crispest RQ4 signal); model-exploitation diagnostics — fraction of planner-selected actions that are invalid, and the model's prediction error on exactly those transitions (an adversarial coverage test of the invalid-action quota).
- **Contingency:** if latents probe well but planning stalls, the diagnosis from the corpus is cost-landscape local minima, not dynamics — apply value-shaping (IQL-shaped distance, arXiv 2601.00844) before concluding failure.

**Exit criteria:** the pre-registered RQ3/RQ4 thresholds in §7, with parser+BFS reported as ceiling context rather than a bar to clear. If neither threshold holds despite good Phase-1 probes, write up the negative result and stop.

## 6. Phase 3 — stretch (only if Phases 1–2 pass)

- **Partial observability:** longer-context predictor (W∈{3,5}, moved here from Phase 1) / recurrent memory; measure how much probing degrades relative to full-obs. The fair nonprivileged classical baseline changes with it: since the JEPA side consumes history, the parser baseline also gets the full action/observation history — a **parser + symbolic belief-state tracker + search**. Deterministic dynamics with known action semantics mean a hand-written tracker can derive everything the history logically implies, which is the same information ceiling the JEPA model faces — so against that baseline the realistic claim is *learned vs hand-engineered*, not superiority. Genuine superiority claims wait for free-form real-shell text, where hand-writing the tracker stops being practical. The goal protocol changes here: a goal *state* can't be rendered through the observation function (observations depend on the last action, and many states render identically), so a goal becomes the expected stdout of a *diagnostic command* (e.g., `cat b/notes.txt` showing content class 3). Known limitation, stated plainly: nothing in this design maintains an explicit belief state over unobserved files — the recurrent context is at best an implicit one.
- **Stochastic outputs:** inject nondeterminism (flaky command, variable output ordering); test the latent-variable-z machinery (VJ-VCR-style) against the deterministic-predictor conditional-mean failure.
- **Free-form actions:** replace the factored vocabulary with real command strings encoded by a text encoder; planner B becomes the only viable planner — which is fine, it's the realistic deployment shape.
- **Real shell:** record trajectories from actual containerized agent sessions; evaluate the world model as a *pre-execution safety/progress scorer* for an existing LLM agent (the practical payoff identified in the corpus: fast what-if evaluation, e.g. previewing `rm`).

## 7. Pre-registered evaluation protocol

Every comparison: ≥5 seeds, arms paired by construction (shared data ordering where applicable, identical held-out episode sets). The **primary gate is a one-sided paired permutation test over seed-level mean deltas at α=0.05** — pairing uses the structure the design already provides, where CI non-overlap would waste it. Hierarchical-bootstrap CIs (resample seeds, then episodes within seeds; seed is the primary unit, since training-run variance dominates with few seeds) are reported for effect-size visualization, not as the gate. Pre-commitment: with 5 seeds the paired permutation test bottoms out at p = 1/32, so any borderline result (0.01 < p < 0.10) triggers a rerun at ≥10 seeds — cheap at this scale — before a pass/fail is declared. Fixed budgets, declared before any experiment: compute-matched training across the JEPA / reconstruction / ablation arms (same trunk, same steps); CEM at 300 samples × 30 iterations (LeWorldModel's setting); K=8 LLM proposals; ≥200 held-out episodes per condition. These numbers are commitments — changing them after seeing results must be disclosed in the writeup.

- **RQ1 pass (Phase 1):** in nuisance regimes, on held-out layouts, **linear-probe** performance (MLP reported separately, never a bar): JEPA ≥ reconstruction twin + 10 points *and* ≥ untrained encoder + 20 points, both absolute, on the **macro average of three per-feature metrics**: cwd accuracy, file-existence balanced accuracy (the bitmap is mostly-absent, so raw accuracy saturates), and content accuracy conditioned on the file existing. The untrained-encoder margin is deliberately larger — random features are a sanity floor, and barely clearing it is itself a red flag. Exact-state match and predicate-satisfaction accuracy are reported alongside as summary metrics, not pass bars. **Baseline pinning (added 2026-07-10, before the slot rerun completed, per adversarial review):** every arm's untrained baseline is *architecture-matched* — same encoder class and output dimension, fresh init per seed — and all artifacts entering a comparison must carry the same probe-protocol version; protocol changes shift the untrained floor by ~3 points, so cross-protocol margins are invalid and baselines are regenerated rather than reused.
- **RQ2 red line (Phase 1):** median banner-swap ‖Δz‖ exceeding 0.5× median state-change ‖Δz‖ means the slow-feature trap dominates — escalate per the risk table before proceeding.
- **RQ3 pass (Phase 2):** *unfiltered* Planner A ≥ *unfiltered* random shooting + 30 points success absolute, at the same sampling budget. The primary pairing is unfiltered-vs-unfiltered because the validity filter helps any planner — an unmatched comparison would credit the parser to the world model. Filtered-vs-filtered is reported as the deployment condition; parser+BFS is reported alongside for context, not as a bar.
- **RQ4 pass (Phase 2):** Planner B beats **both components alone** under the primary paired test: (a) the greedy LLM, and (b) Planner A (its best variant) — each at its declared budget, since losing to pure JEPA search at deployment budgets means the LLM proposal stage subtracts value and RQ4's answer is no. A matched world-model-rollout comparison (CEM budget clamped to Planner B's K·H rollouts) is reported descriptively. Additionally, JEPA ranking must change the LLM's choice in ≥10% of episodes with net-positive effect.

## 8. Risks

| Risk | Signal | Mitigation / interpretation |
|---|---|---|
| Slow-feature trap dominates | Banner identity decodable from z while state isn't; z moves more on banner swap than on state change | Enable the IDM arm (2211.10831 found inverse dynamics robust to distractors); make banners vary *within* trajectories (a fast distractor is no longer slow); document as a text-domain replication of 2211.10831 |
| JEPA ≤ reconstruction baseline on probes in nuisance regimes | RQ1 fails | Raise the noise knobs (more banner entropy, heavier dynamic noise); if JEPA still can't win where there is genuine nuisance to abstract away, the premise doesn't hold at this scale |
| Good latents, failed planning | RQ3 fails with RQ1 passing | Cost landscape, not dynamics → value shaping (2601.00844) |
| SIGReg weak under low data diversity | Collapse-adjacent metrics (embedding rank/variance) sag | Known LeWorldModel limitation; widen layout distribution in datagen |
| LLM already plans this domain perfectly | Greedy LLM ceiling on toy tasks | Likely, since Phase 2 is full-obs: raise combinatorial depth (multi-predicate goals with ordering/interference constraints, larger trees); if the ceiling persists, RQ4's decisive comparison moves to Phase-3 partial obs — report honestly if the hybrid adds nothing in full-obs |

```

  env/            # sandbox FS world, action defs, renderers (full/partial obs), distractor knobs
  datagen/        # typed action sampling (validity quotas), transition-type labels, splits (held-out predicates + layouts)
  models/         # obs encoder, compositional action encoder, predictor, SIGReg/VICReg/IDM/temporal-sim losses
  baselines/      # reconstruction (generative) twin, parser + BFS/A* classical planner
  train/          # training loop, rollout-loss schedule, collapse metrics logging
  probes/         # linear/MLP probing harness against env ground truth
  plan/           # CEM planner (± validity filter), LLM-propose/JEPA-rank planner (grammar + lowering), plan-time goal-exemplar construction
  eval/           # RQ1–RQ4 report generation
```
