# Terminal-JEPA

Testing whether Yann LeCun's **JEPA** world-model recipe transfers to a **real-shell agent**: a sequence world model over real Linux **Docker** filesystems that, given an exploration history, predicts command outcomes on **unseen systems** — improved by a Claude-Code-native **ShinkaEvolve** evolutionary search. This is the project's front page and **living status** (overview + current results); it leads with the active R4→R7 work, with the retired synthetic phase compressed to "prior work" below.

Last updated: 2026-07-18. Companion docs: [JEPA.md](JEPA.md) (background: LeCun's JEPA research program, 2022–2026), [ShinkaEvolve.md](ShinkaEvolve.md) (the evolutionary-search method) and [terminal-jepa/evolve/CLAUDE.md](terminal-jepa/evolve/CLAUDE.md) (**the evolve loop's working context + replication manual — read this to rerun/iterate the search**). Code lives in [terminal-jepa/](terminal-jepa/) with its own README (module inventory, commands).

## Where we are

The project is a **real-shell JEPA world model** over Docker filesystems, improved by a Claude-Code-native **ShinkaEvolve** evolutionary search. Over real Linux images, given an exploration history, the model predicts command outcomes on **unseen systems** — beating lexical memory, honest baselines, and a generative twin (the R4 "first clearly positive JEPA-aligned result" in the program). The toolset (`uname`/`cat`/`ls`/`cd`) is a deliberately small navigational **starting point**, meant to expand — not a fixed scope. The evolutionary search then roughly doubled the held-out margin: **held-out final-test content-verb margin 0.306 → 0.5702** (content top-1 0.31 → 0.86; R4→R7), every champion validated (latent ≫ generative twin; history-driven; gains hold on novel context). The winning designs came disproportionately from **cross-domain inventor lenses** (a hippocampal episodic-memory arch; a delta-rule fast-weights arch; a free-energy precision objective).

## Phase R4 — real Docker filesystems, exploration sequences, held-out images (the foundation)

Substrate (`realenv/docker_env.py`, `realenv/collect_docker.py`): real commands run inside real Linux Docker images (distros + databases + web servers + runtimes), so filesystems have real common paths, real system identity, real tool output. Each trajectory is a **sequence** — identify the system (`uname` + `cat` a config file), then navigate/inspect (`cd`/`ls`/`cat`) over the image's *real* paths — so the world model must predict later observations from the **accumulated history** ("if I `uname` then `cd /etc`, what's there?"). Toolset (complementary/navigational): `uname`, `cat` (system config files), `ls`, `cd` — a deliberately small **starting set**; expanding it (e.g. `man`/`less`/other tools) is future work, and the content-verb eval grows with it. **Dataset (`data/dockerfs/`): ~77K real steps** — train 51,190 (8 images: alpine/ubuntu/debian/python/redis/nginx/postgres/node) + val **25,660 on 4 HELD-OUT image types** (fedora/rocky/mariadb/httpd). **The fair test:** predict `ls`/`cat`/`cd` outcomes on **unseen paths/systems** from an exploration history, beating predict-mean and a generative twin. Data note: in Docker `uname`'s kernel line is the shared host-VM kernel, so system identity lives in `/etc/os-release`/filesystem layout, not `uname`. **Built:** the **sequence world model** (`realenv/seq_worldmodel.py`) — a causal transformer over the interleaved `cmd,obs,cmd,obs,…` stream of frozen-ModernBERT embeddings; the hidden state at each **command** position predicts that command's resulting **observation** embedding (latent, standardized). Only the transformer is learned. The eventual goal (read a man page and infer a *new* tool) is explicitly deferred, not an initial gate.

**R4 result — the first clearly positive, JEPA-aligned result in the program (3 seeds).** On **held-out images** (unseen systems fedora/rocky/mariadb/httpd), evaluated by next-observation **retrieval** (rank the true next obs against 63 foils by latent distance; top-1, chance = 1/64 = 0.016; hard **same-verb** foils):
- **The world model beats every honest baseline on the content verbs.** ls+cat (outputs that are *not* echoable from the command; ~65% of steps): **WM top-1 ≈ 0.56** vs lexical **retrieve-by-command 0.25**, history-free MLP 0.21, copy-prev 0.10, **predict-mean 0.02 (≈chance ✓ calibrated)**. Per verb: ls **0.54** (lexical 0.16), cat **0.58** (lexical 0.36). Latent MSE 0.32 vs predict-mean 0.97; cosine-to-true 0.82. (The overall top-1 incl. verbs, 0.639, is inflated by `cd`, whose observation is just `cwd=<target>` — trivially echoable — so the honest headline is the content-verb number.)
- **The exploration history is the driver, not model capacity.** Matched-capacity control (the *same* transformer with self-only attention — identical architecture/depth/params, differing only in whether command positions may attend to the history): content-verb top-1 **full 0.558 vs masked 0.296 → history gain +0.262 ± 0.005**; ls +0.32, cat +0.19, and **cd +0.00** (echoable, needs no history). The gain appears exactly where a real world model predicts and is absent where predicted.
- **JEPA (latent-prediction) beats the compute-matched generative twin** (token-bag reconstruction), each scored by the same retrieval metric in its own native space: **0.64 vs 0.53** (+0.11). Reported *with a caveat* — the two live in different geometries (768-d MSE vs 4000-d BCE), so this is suggestive, not a clean apples-to-apples objective comparison; the generative twin is a legitimate non-strawman (0.53 ≫ chance).
- **In-distribution reference:** dev (seen images, unseen sequences) top-1 = 0.855 vs held-out 0.639 — a real but healthy generalization gap.
- **What this does and does NOT show.** Shows: over real Docker filesystems, a JEPA-style latent world model, given an exploration history, predicts the latent consequence of `ls`/`cat`/`cd` on **unseen systems** better than lexical memory, honest baselines, and a generative twin. Does NOT show: inferring an unseen *tool* (deferred by design); *generating* the literal output text (this is latent retrieval, the JEPA bet, not reconstruction); closing the dev→held-out gap; and part of the ls/cat signal is legitimate shared-filesystem-structure generalization across distros (common paths), which the lexical baseline already banks and the WM beats. Guardrails: `tests/test_seq_worldmodel.py` (no-future-leakage regression, self-only invariance, retrieval calibration). Artifacts: `runs/dockerfs/{seq-worldmodel,seq-history-ablation}.json`.

## Phase R5–R7 — the ShinkaEvolve evolutionary design search on the R4 foundation (`evolve/`)

A Claude-Code-native ShinkaEvolve loop: decompose the world model into swappable **chunks**, have heterogeneous inventor subagents (**Claude** via the Workflow tool at effort=high with mandatory code-reading + **Codex** via `codex exec` — real model diversity, with arxiv/cross-domain research encouraged) invent/recombine chunk implementations, score by a trustworthy **content-verb-margin fitness** (reusing the R4 eval) with per-genome no-leakage + calibration hard filters and a mandatory **3-way split** (fit / inner-val fedora+mariadb / untouched final-test rockylinux+httpd), cheap proxy → full promotion. **Read [`evolve/CLAUDE.md`](terminal-jepa/evolve/CLAUDE.md) for the full replication manual** (chunks, contracts, contract extensions, fitness, the proxy→full ladder + dev-environment-nondeterminism caveat, the inventor-dispatch pattern, recombination/epistasis, the round recipe). Full per-genome stats in the `evolve-insights` auto-memory note + `evolve/archive/genomes.jsonl`.

- **The headline: held-out final-test margin 0.306 → 0.5702 over R5–R7** (content top-1 0.31 → 0.86; full budget, 3 seeds, untouched rockylinux+httpd). Progression:

  | step | change (inventor lens) | full-final margin | wm content top-1 |
  |---|---|---|---|
  | R4 baseline | MSE + ModernBERT | 0.306 | 0.31 |
  | R5 objective | + L2-InfoNCE (Claude) | 0.459 | 0.72 |
  | R6 encoder | + e5-base-v2 retrieval-tuned "eyes" | 0.5017 | 0.80 |
  | R6 recombine | + recency-ALiBi arch / focal obj / warmup optim (stacked) | 0.5471 | 0.84 |
  | R6 arch | + hippocampal episodic-memory arch (neuro lens) | 0.5607 | 0.85 |
  | R6 batcher+obj | + system-blocked hard-neg batcher + free-energy precision obj | 0.5641 | 0.857 |
  | **R7 arch** | **+ path-delta fast-weights arch (Codex)** | **0.5702** | **0.863** |

  **Current best genome** (`r7-arch-fastweights`): e5-base-v2 encoder (`data/dockerfs-e5`) + `r7_path_delta_fastweights_codex` arch + `r6_free_energy_precision_l2_contrastive` objective + `r6_sysblock_hardneg_curriculum` batcher (block2/hard75/ramp30) + `go_warmup_holdcos_floor` optim. Four of its five components are evolved (only the optimizer is a hand-designed schedule); three came from cross-domain inventor lenses.

- **The eight evolvable chunks (axes).** `objective`, `arch`, `optim`, `target`, and — added during R6/R7 as new evolvable axes, each with a bit-identical baseline that reproduces prior behavior exactly — `perception` (the encoder "eyes" + render + pooling, scored via a data-side re-encode), `batcher` (training-batch composition = the in-batch negative pool), `stream` (single- vs multi-vector token layout), and `head` (the readout module + optional aux task). Two harness *contract extensions* were needed and are load-bearing: **LEARNED targets** (a target module trained jointly, evaluated via its exact inverse in the fixed obs space) and the **head axis** (a wrapper that re-points `net.forward`, requiring care against a parent↔child module cycle and forward recursion). Details in `evolve/CLAUDE.md`.

- **Recorded chunk results (content-verb margin; proxy = 1000 steps/seed 0, full = 4000/3 seeds; the neutral per-genome ledger is `evolve-insights`).** Highlights by axis:
  - *objective:* mse 0.306 final → L2-InfoNCE 0.459 → focal-listwise/sameverb ~0.461–0.463 → **free-energy precision-weighting 0.5641** (the current objective). Failed/neutral: additive-margin, NCA-hinge (negative), Sinkhorn assignment (0.34 proxy), manifold-capacity and local-geometry (below), and fusing hubness onto free-energy (fe-mutprox) was neutral-to-negative.
  - *arch (the biggest lever):* baseline transformer → recency-ALiBi / sysid / two-stream routing (each ~+0.005–0.01, and they **stack**) → **hippocampal episodic-memory read 0.5607** (a content-addressable convex combination of past observation embeddings, gated into a transformer) → **path-delta fast-weights 0.5702** (a delta-rule associative memory that writes each cmd/obs pair into two online memories keyed by command content and by path/context, read back in target space at future command positions). The R7 unification/attractor variants (CA3 iterative read, theta-sweep rollout) were within proxy noise of the leader.
  - *perception (the biggest single data-side lever):* retrieval-tuned **e5-base-v2 = +0.043 final** over ModernBERT (first genome > 0.50); bge < e5; codebert (code-aware, not retrieval-tuned) < ModernBERT — so retrieval-tuning is the helpful property. **e5-large *lost* margin** despite comparable wm top-1 because it lifted the objective-independent baselines as much as the WM (the margin rewards *differential* help, not absolute retrievability). Asymmetric query:/passage: prefixes ≈ tied; predictive-instruction query and foveal renders below.
  - *batcher:* the system-blocked hard-negative **curriculum beats uniform** (densifies same-system in-batch negatives to match the eval's same-verb-same-system foils); harder blocking trended up; (verb,system)-collision-clique variant below.
  - *optim:* warmup→hold→cosine-to-floor best; SWA-tail-averaging and a CLS two-timescale trust optimizer below.
  - *target:* **every pure transform lost to identity** (partial-residual, delta, Householder-frame, radial companding); learned target-space transforms (diagonal gate, off-diagonal Givens rotation) came within noise of identity — the target space is hard here, plausibly because the free-energy objective already does the per-dim reweighting a diagonal target would add.
  - *stream:* both multi-vector readouts tried (R6 line-strips, R7 role-canonical entry-names+metadata) **failed to beat their matched single-vector controls** at proxy.
  - *head (new R7):* no gain (mutation below its clean-isolation control; the head bypasses the hippo episodic read, so it was isolated on a `pred=head(h)` arch).
  - *exploration:* a Lévy-walk + count-based-novelty collection policy scored below the baseline-policy train set even with the verb-mix confound removed (a residual path-statistics train/val mismatch remains).

- **Champion validations (final-test, 3 seeds).** Every promoted champion cleared the same arms that certified R4. For the R7 best (`r7-arch-fastweights`): gen-twin latent-prediction **0.883 vs generative-reconstruction twin 0.577** (+0.31 — the JEPA bet holds); history ablation full **0.836 vs masked 0.439** (history gain 0.398). For the R6 best, an exact-genome history ablation (full genome vs a matched-capacity self-only + memory-off control) gave a history gain of 0.42–0.45, larger than the baseline-arch gain, and a revisited-split showed the memory-arch gains hold on **novel-context** steps (~95% of the eval), not just repeated commands.

- **Process facts (methodology, verified).** (1) **Inventor effort is load-bearing:** at default Agent-tool effort no inventor beat the incumbent; at Workflow effort=high with mandatory code-reading, grounded inventors did — so the round pattern is high-effort code-grounded Claude inventors + Codex for model diversity. (2) **MPS proxy nondeterminism is ~±0.001** for the identical genome (CPU training is bit-exact), so proxy deltas < ~0.002 are noise and full-budget 3-seed results are the reliable signal. (3) **Epistasis is tested, never assumed:** marginal per-chunk wins *stacked* (positive) in R6; in R7 an objective that helped on one arch *hurt* on the fast-weights arch (negative), so it was dropped. (4) **Failed traits are retried recombined** — the target-space idea failed 3× early, then came within noise of identity in the e5+contrastive context. (5) Bit-identical **plumbing checks** gate every new axis; inventor integration bugs (a module cycle, a Codex `torch.where` crash) were hardened/salvaged in-loop. (6) Culture: **record stats, not verdicts** — the ledger stays neutral so it doesn't bias the next round.

## Prior work: the synthetic negative and the real-shell prototypes (why we pivoted)

The retired substrates and their code were removed 2026-07-18 (git history retains everything). Both are honest, adversarially-reviewed negatives — they are why the current framing (real systems, shared toolset, held-out *paths/systems*, world-model quality via evolution) exists.

- **Synthetic filesystem sandbox (Phase 0–1, findings 1–24).** A deterministic in-process FS world with a 301-slot hand-coded ontology and factored `(verb, arg1, arg2)` actions, testing whether JEPA latents linearly decode true state and transfer to held-out *layouts*. Result across many recipe variants (SIGReg/VICReg/IDM/temporal-sim; CLS and slot latents; scale, data diversity): **no variant transferred state to held-out layouts** — at artifact scale, trained encoders decoded held-out layouts *worse than architecture-matched random features* (the sharpest form of the layout-entanglement problem). A frozen-pretrained-encoder + symbolic-decode pipeline measured near-oracle, but contained **no learned dynamics over frozen features** — a pretrained-initialized predictor over frozen features failed the dynamics gate, and **JEPA never beat the compute-matched generative twin**. No gate in the pre-registered ladder passed as written. The wall: compositional binding over a non-canonical representation, and it looked modality-independent.
- **Real-shell prototypes (R1–R3).** Rebuilt on real containerized shell sessions with a held-out-*tool* bar (predict `git`/`sed`/… having never seen them). The world model learned genuine dynamics on trained tools but **did not transfer to unseen tools** (lost to predict-mean at 5 seeds), and JEPA again did not beat the generative twin (10 seeds) — replicating the synthetic negative at scale. **Milestone correction (the pivot to R4):** inferring an unseen tool's syntax is a read-the-man-page capability no agent could do cold — an *eventual* goal, not an initial gate. The right bar is held-out **paths/systems** with a small shared toolset over real filesystems, which is R4 above.
- **What carried forward** (so the negative wasn't wasted): the evaluation rigor — honest baselines, multi-seed, architecture-matched controls, adversarial review before write-up, "record stats not verdicts" — is now the evolve **3-way split + guardrails**; the finding that a **frozen retrieval-tuned encoder** is the strongest backbone became the R6 `perception` chunk (the biggest data-side lever); and the confounded-target lesson (the command echoes into the observation) shaped the R4 content-verb eval.
