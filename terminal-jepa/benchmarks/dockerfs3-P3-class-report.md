## 6. Surprising cells / potential honesty holes

1. **`grep|hit` excluded as semi-echo under the 0.656 axis-2‚Ä≤ PRIOR ‚Äî but ¬ß2 item 3 says linked-grep-hits belong on the CONTENT side.** `grep|hit|native` a2‚Ä≤=0.847, `grep|hit|mutated` a2‚Ä≤=0.927. These are exactly the "linked-grep hits = content side" anchor cases the axis-2‚Ä≤ freeze is supposed to keep as content. With the provisional 0.656 prior they are wrongly excluded. **The pilot hands the freeze its calibration target:** the max-margin midpoint must sit *above* linked-grep-hits (‚âà0.93) and *below* the observed-capture echo-loop readbacks (`cat|hit|created-obs`=1.000, `readlink|hit|native`=1.000) ‚Äî i.e. **axis-2p_thresh ‚âà 0.96**. At that threshold `grep|hit|native`‚Üícontent, `grep|hit|mutated`‚Üícontent, while the true echo-loop readbacks stay semi-echo. This is the single most consequential finding for the class freeze. *(Until recalibrated, grep is missing from the content surface ‚Äî a real gap, not a leak.)*

2. **`cat|miss|mutated` ‚Üí sim (a3ex=0.935, 186 steps).** After a mutation removes a file, `cat` of it yields a deterministic dialect error the SST reproduces from P0 error templates ‚Üí correctly excluded as `sim`. Not a hole; it shows the SST error surface doing real work on a large slice. (It is miss-mode, so never a content candidate.)

3. **`cd|ok|native` ‚Üí semi-echo via axis-2 PREFIX containment (0.711), not axis-1 (which is 0.538, below Œ∏1).** Confirms the resulting-cwd render-prefix column is load-bearing: without it cd's cwd-echo would slip past rule 1 and (barely) past rule 3. Working as designed.

4. **`ls|hit|created` ‚Üí ack (a3ex=1.000, dup=1.0, median 4 chars), coverage exactly 30.** An `ls` of a freshly-created empty dir is a near-empty deterministic listing ‚Üí ack (rule 2). Borderline coverage; SST-determined either way, so excluded correctly, but a cell to watch (its class could wobble ack‚Üîsim at scale).

5. **Metadata quirk:** `classes.json` `encoder`/`twin_root` fields read `custom`/`null`. Axis-1 really used the frozen e5 encoder; axis-4 really has no twin (ceiling‚â°1.0). Documented in the draft's `_pilot_provenance`. Worth a one-line fix in class_measure so the frozen file's provenance is self-describing, but it does not affect any class.

6. **43/69 cells under-floor (451 steps, 10.0% of all steps) ‚Äî fail-closed, as designed.** These are the mutated/created sub-cells and every composed family, all starved at 40 seqs√ó4 images. None were guessed; each will clear coverage at mint scale. The under-floor set is where the composed-content and blind-created content mass is currently hiding.
