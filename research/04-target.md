# Designing the Prediction Target in a JEPA-Style Latent World Model

*Research note for the terminal-JEPA program — the `target` chunk*

## Abstract

The prediction *target* — the quantity a latent world model is trained to reproduce at each step — is one of the most consequential and least glamorous design choices in a Joint-Embedding Predictive Architecture (JEPA). This note surveys the main theoretical families for shaping that target: predicting an abstract embedding rather than raw pixels/tokens (the core JEPA argument); the target-*encoder* design (EMA/momentum teachers, stop-gradient) and why an asymmetric, slowly-moving target is what prevents representational collapse; target normalization, whitening, and decorrelation as target-space shaping (Barlow Twins, VICReg, W-MSE); residual/delta/velocity targets from sequence and diffusion/flow modeling; and learnable invertible (rotation/companding) target transforms together with the identifiability hazards they raise. We connect each family to what terminal-JEPA's own `target` search found: identity (predict the next embedding directly) is a hard baseline; every *pure* transform (delta, Householder, radial companding) lost to it; and *learned* diagonal/Givens transforms tied identity within measurement noise — consistent with the hypothesis that the model's free-energy objective already performs the per-dimension error reweighting a diagonal target would supply.

## 1. The role of the target in this world model

In terminal-JEPA, the substrate is a causal transformer over an interleaved `cmd, obs, cmd, obs, …` stream of *frozen* encoder embeddings; the hidden state at each command position is trained to predict that command's resulting observation embedding, and the retrieval eval ranks the true next-observation embedding against hard same-verb foils by squared L2 in a fixed observation space. The `target` chunk is the single seam that decides *what* the predictor regresses toward. Its contract is deliberately narrow: a pure impl exposes `make_target(z_obs, z_prev)` (the tensor the model is trained to predict) and `to_obs(pred, z_prev)` (a reconstruction of the predicted next-obs embedding for the eval), where `z_prev` is the causally-available previous observation, so no future information leaks. A `LEARNED` impl instead returns an `nn.Module` trained jointly with the predictor, and the harness evaluates it **through its exact inverse in the fixed observation space**.

That exact-inverse rule is the honesty mechanism, and it deserves emphasis because it is what makes the target axis safe to search. A learned target space is free to reshape the geometry the loss sees, but it can never move the geometry the *eval* sees: the retrieval metric always ranks the true `z_obs` against foils in the original obs basis, and a prediction is scored only after `to_obs` maps it back. A target that "cheats" by shrinking a dimension to dodge its prediction error simply re-amplifies that error on reconstruction; a collapsed learned target (all inputs to one point) cannot reconstruct varied targets and is scored down. Because `to_obs(zeros) == zeros` for every impl, the predict-mean calibration guard is unaffected. This is the local analogue of the JEPA collapse problem (§3): a target space, like a target encoder, can trivially satisfy a naive objective, and the eval must be constructed so that the trivial solution loses.

## 2. Predict an abstract target, not the raw input

The founding JEPA claim is about *what space* the target lives in. LeCun's 2022 position paper argues that generative, input-space prediction forces a model to commit to unpredictable detail — the exact texture of grass, the exact ripples on a pond — and "cannot eliminate irrelevant details other than by pushing them into a latent variable," whereas a JEPA's target encoder can abstract those details away [1]. The formal reading (Dawid & LeCun) is energy-based: the training energy is the prediction error in embedding space, `E = dist(s_y, Pred(s_x, z))`, with `z` an optional latent for what is genuinely unpredictable [2]. I-JEPA instantiated this at scale on images — predict the *representations* of masked target blocks from a context block — and reported strong semantics without hand-crafted augmentation and roughly 5× fewer iterations than pixel-reconstruction (MAE-class) methods [3]. V-JEPA's controlled ablation is the cleanest single data point: latent-space feature prediction beat pixel-space prediction by about +5 points on frozen Kinetics-400 evaluation, all else equal [4]. For terminal-JEPA the modality is shell output, but the bet is identical and already validated at the substrate level: latent next-observation retrieval beats a compute-matched generative (token-bag reconstruction) twin. The `target` chunk operates entirely *within* this abstract space; it never reintroduces input-space reconstruction, only reparametrizes the embedding the predictor aims at.

## 3. The target-encoder design: why asymmetry prevents collapse

If both sides of a joint-embedding objective are trained to agree, the trivial solution — every input maps to the same constant vector, error zero — is always available. Preventing this *representational collapse* is the central engineering problem of the family, and the dominant solution is architectural asymmetry in how the *target* is produced.

- **Momentum / EMA targets.** MoCo introduced a momentum-updated key encoder (a slow exponential moving average of the query encoder) to keep a large, *consistent* dictionary of targets during contrastive learning [5]. BYOL then showed you can drop negatives entirely: an online network predicts the representation of a *target* network whose weights are a slow moving average of the online network, and this asymmetry alone is enough to avoid collapse — 74.3% ImageNet linear top-1 with no negative pairs [6]. DINO recast the same idea as self-distillation with no labels, with a momentum *teacher* plus centering/sharpening to prevent collapse [7], and DINOv2 scaled that recipe into general-purpose frozen features [8]. I-JEPA and V-JEPA carry the pattern into predictive architectures: the target encoder is an EMA copy of the context encoder (momentum ≈ 0.996), receives no gradient, and produces the semantic targets the predictor regresses toward [3, 4].
- **Stop-gradient.** SimSiam isolated the essential ingredient: a Siamese network with *no* momentum encoder, no negatives, and no large batches still learns — provided a **stop-gradient** on the target branch; remove it and the model collapses immediately [9]. Tian, Chen & Ganguli's dynamics analysis (and their gradient-free DirectPred predictor) explains *why*: the extra predictor plus stop-gradient shape the learning dynamics so the predictor's eigenspectrum tracks the correlation structure of its inputs, steering the system away from the collapsed fixed point [10].
- **Frozen targets.** When the target encoder is a frozen pretrained model, collapse is impossible by construction because the targets cannot move. This is exactly terminal-JEPA's regime: the ModernBERT/e5 encoder is frozen, so the observation targets are fixed points and the classic EMA/stop-gradient machinery is unnecessary. data2vec is the general-modality relative — predict the *contextualized latent representations* of a masked input from an EMA teacher, unifying speech/vision/language — and shows the target-representation idea is modality-portable [11].

The lesson terminal-JEPA inherits is subtle: because its encoder is frozen, the collapse risk does *not* live in the target encoder — it re-enters only when a `LEARNED` target module is allowed to reshape the space the loss sees. The exact-inverse eval (§1) is the frozen-target discipline applied one level down: it plays the role the stop-gradient/EMA asymmetry plays upstream, guaranteeing a degenerate target transform cannot win.

## 4. Target normalization, whitening, and decorrelation

A second family shapes the target *distribution* rather than its encoder. These methods can be read as target-space preconditioning — deciding which directions of the embedding the loss should treat as signal.

- **Standardization.** Terminal-JEPA already standardizes observation embeddings per dimension (the retrieval metric and the `sqrt(D)` shell in the companding impl both assume `E‖z‖² = D`). This is the mildest target shaping and is upstream of every `target` impl.
- **Decorrelation objectives.** Barlow Twins drives the cross-correlation matrix between two views' embeddings toward the identity — diagonal terms enforce invariance, off-diagonal terms *reduce redundancy* between feature dimensions — and thereby avoids collapse without negatives [12]. VICReg makes the two forces explicit as separate loss terms: a variance hinge that keeps each dimension's std above a floor (the anti-collapse term) plus a covariance term that decorrelates dimensions [13].
- **Whitening.** W-MSE whitens each embedding batch (ZCA/Cholesky) before the MSE, scattering samples across the space so that collapse is impossible and negatives are unnecessary [14]. A later systematic study found that simply appending ZCA whitening as the encoder's last layer improves linear/kNN probing by 1–5% across many SSL methods and backbones — evidence that decorrelating the target space is broadly, if modestly, helpful [15].

The connecting idea is that the "right" target metric is not the raw Euclidean one but a *whitened* one: `Σ^{-1/2}`-transformed error, so correlated nuisance directions are down-weighted. This is precisely the theoretical motivation the project's learned-rotation `target` impl invokes — training in a rotated basis makes the effective error metric `Qᵀ diag(Π) Q`, a full (non-diagonal) precision matrix of ZCA-whitening form, which a diagonal reweight cannot express (the impl cites the whitening-improves-SSL result [15]).

## 5. Residual, delta, and velocity targets

A third family changes *what quantity* is predicted: the change rather than the absolute state. The intuition is variance reduction — if consecutive states are similar, the residual is lower-variance and easier to regress. terminal-JEPA's `delta_prev` impl is the canonical form: `make_target = z_obs − z_prev`, `to_obs = z_prev + pred`. On a shell trace many commands leave the observation close to the previous one (same cwd banner, an `ls` of a nearby directory), so this is a reasonable prior.

The same reparametrization is load-bearing elsewhere. In continuous generative modeling, **flow matching** trains a network to regress a *velocity field* — a time-derivative/delta — that transports a base distribution to the target, rather than the absolute sample [16]. Diffusion's **v-parameterization** (progressive distillation) predicts a velocity `v = α_t ε − σ_t x` — a rotation of the (noise, signal) target pair — specifically because that reparametrized target is numerically more stable at few steps than predicting the raw sample or the raw noise [17]. Both are concrete evidence that *which affine/derivative form of the target you regress* changes optimization behavior even when the underlying task is fixed. The catch, visible in both literatures and in terminal-JEPA's result, is that residual targets only help when the residual is genuinely lower-variance *and* better-conditioned than the absolute target; if the reconstruction step re-injects the removed component (as `to_obs = z_prev + pred` does), a poorly-predicted residual is no easier to rank than the absolute embedding. In this project `delta_prev` and the Householder-canonical residual (which reflects `z_prev` onto one axis to concentrate the "same as previous" component, keeping the transform exactly orthogonal and invertible) both *lost* to identity.

## 6. Learnable invertible target transforms and identifiability

The most ambitious family makes the target transform itself *learned* — a jointly-trained, invertible map of the target space. terminal-JEPA explored three, each identity-at-init and exactly invertible so the eval can undo them:

- **Diagonal gate** (`tgt_space_diag_gate`): `g = D·softmax(θ)` (mean-one per-dim gates), `make_target = g·z_obs`, `to_obs = pred/g`, with a KL-to-uniform stabilizer. This lets gradient down-weight dimensions whose variation is unpredictable noise.
- **Givens rotation** (`r7_learned_givens_rotation_decorrelate`): a composition of Givens-rotation layers over fixed random coordinate pairings — exactly orthogonal for any angles (so `to_obs = Qᵀ pred` is the exact inverse), dense after several layers, identity at init. Its stated purpose is to supply the *off-diagonal* basis rotation a diagonal precision cannot represent, composing into a full ZCA-style precision metric [15].
- **Radial companding** (`r10_radial_power_compand_sched`): a scheduled, exactly-invertible pure-radial bijection around the `sqrt(D)` standardization shell, aimed not at ranking but at *calibrating* prediction norm (the contrastive objective is invariant to a prediction's own norm, so the trunk drifted to `‖pred‖²/‖true‖² ≈ 4.7`, off-manifold — the companding inverse maps that inflation `k → k^{1/s}` back toward the shell).

Two theoretical hazards govern this family. First, **identifiability / degeneracy**: a freely learnable target transform can lower the *training* loss by warping the space rather than by predicting better, up to and including collapse. The exact-inverse eval is the guard — because ranking happens in the fixed obs basis and orthogonal/bijective maps preserve (or exactly restore) the positive-pair distance, a degenerate transform gains nothing and a norm-shrinking transform is penalized on reconstruction. This is "honest by construction," and it is what let the project search this axis without corrupting fitness. Second, **redundancy with the objective**: a diagonal target gate is the *same kind of operator* as the free-energy objective's diagonal precision weighting `Π_d = 1/MSE_d` in the fixed basis; the two commute and are redundant, so the gate can only add what the objective already adds — predicting a flat result. This is exactly the reasoning that motivated escalating from the diagonal gate to a *rotation* (the operator a diagonal weighting cannot express) and then to a norm-only companding (orthogonal to the ranking objective entirely).

## 7. What terminal-JEPA found — and why identity is hard to beat here

The recorded `target` results are uniform and instructive (content-verb margin; proxy = 1000 steps/seed 0, full = 4000/3 seeds; dev-environment proxy noise ≈ ±0.001):

- **Every *pure* transform lost to identity.** Partial-residual, `delta_prev`, the Householder-frame residual, and radial companding all scored at or below the identity baseline. The residual prior (§5) did not pay off: shell observations are not reliably close to their predecessor once `cd`/system boundaries are crossed, and the reconstruction step re-injects `z_prev`.
- **Learned target-space transforms tied identity within noise.** The diagonal gate came in at roughly −0.002 (within noise), and the off-diagonal Givens rotation likewise landed within the noise band of identity. The companding impl was a *calibration* intervention, not a ranking one: it demonstrably pulls prediction norm back onto the manifold (toy `k` from 4.78 → ~1.46 at `s=6`) at a small fitness cost, but did not lift the fitness margin.
- **The most likely explanation is interaction with the objective.** The champion objective is a free-energy precision-weighting loss that already reweights per-dimension errors by their inverse variance — i.e. it already does, inside the loss, the diagonal reshaping a diagonal target would add. A diagonal target is therefore redundant (§6); an orthogonal target would need to find a *correlated* nuisance mode the precision weighting misses, and apparently the gain from doing so is below the ±0.001–0.002 detectable band on this data. This is a clean instance of epistasis: the target axis is not independently valuable *given* an objective that already preconditions the error metric.

The broader-program context makes identity's strength unsurprising. The frozen retrieval-tuned encoder already produces a well-conditioned, standardized target space; the collapse pressure that motivates elaborate target-encoder machinery upstream is absent here; and the JEPA-family evidence that *target-space* choices matter (whitening's +1–5% [15], v-prediction's stability [17]) is real but *modest* in magnitude — exactly the size of effect that a ±0.001 noise floor and a redundant objective can wash out.

## 8. Design guidance and open questions

- **Treat identity as the null and demand a mechanism the objective cannot already supply.** A diagonal target under a precision-weighting objective is a priori redundant; only *off-diagonal* (rotation/whitening) or *non-metric* (norm-calibration) structure can add anything, and only if the data actually contains a correlated nuisance mode.
- **Retry the target axis in changed contexts.** The target idea failed three times early, then came within noise of identity once recombined with the e5 encoder and contrastive objective. Whether a learned rotation helps is contingent on the objective and encoder; it should be re-tried whenever those change (e.g. paired with an objective that is *not* already precision-weighted, where a whitening target might carry real signal).
- **Separate ranking from calibration.** The companding result shows the two objectives are distinct: a target transform can be worthless for fitness margin yet valuable for downstream latent-MPC (which needs on-manifold norms). Future `target` impls aimed at planning should be evaluated on calibration/planning metrics, not the content-verb margin alone.
- **Keep the exact-inverse guard sacred.** It is the single reason a learned target space is safe to evolve; any new impl must remain exactly invertible (or provide `reg()` that cannot be gamed) and preserve `to_obs(zeros)==zeros`.
- **Open questions.** Does a *stochastic* target (a latent-variable `z` for genuinely multimodal command outcomes, as in the position paper's EBM framing [1, 2]) help where deterministic residuals do not? Would a whitening target trained against a plain-MSE objective (removing the redundancy with precision weighting) finally clear identity? Can a target-space transform that is learned *per-verb* rather than globally exploit the block structure the batcher already targets?
