# The `arch` Chunk: Architectural Theories for a Causal Sequence World Model

*A research note for the terminal-JEPA program. Scope: the neural-network design space for the sequence backbone that accumulates a shell-exploration history and predicts the latent consequence of the next command, under a hard no-future-leakage constraint.*

## Abstract

The `arch` chunk is the learned core of the terminal-JEPA world model: a causal network that consumes an interleaved `cmd, obs, cmd, obs, …` stream of frozen 768-d embeddings and, at each command position, emits a prediction of that command's next-observation embedding using only strictly-past tokens. This note surveys the main theoretical frameworks for designing such a network — self-attention Transformers and their positional/recency schemes; recurrent, linear-attention, and state-space alternatives under the associative-memory view; fast-weight programmers and the delta rule; modern Hopfield and hippocampal/complementary-learning-systems models of episodic memory; and recurrent state-space backbones from model-based RL — grounding each family in its primary literature and its mechanistic reason for working. It then connects these frameworks to the project's own measured arch progression (baseline Transformer → recency-ALiBi/system-id → hippocampal episodic read → path-delta fast-weights → chunkwise-WY delta memories, the current champion and also 6.95× faster than its parent), which recruited its two largest wins from the neuroscience and delta-rule lenses. A recurring empirical caution is documented: proxy→full ranking is *not* consistent across arch families, so mechanism-level reasoning must be validated at full budget.

## 1. What the arch must do (the contract)

The arch contract is deliberately narrow (`evolve/CLAUDE.md`): an impl exposes `build(**p) -> nn.Module` whose `forward(tok_emb [B,L,768], types [B,L], key_pad [B,L]) -> (pred [B,L,768], h [B,L,d])` predicts at *every* position, and the harness reads command positions `pred[:, 0::2]` as the next-observation predictions. Two structural facts follow.

**Causal history accumulation is the whole point.** The frozen encoder ("perception") turns each command and each observation into a fixed embedding; the arch is the only learned dynamics. Because the R4 result showed the *exploration history is the driver, not model capacity* (content-verb top-1 0.558 with history vs 0.296 self-only), the arch's job is to build, at command position `2t`, a state that integrates `obs_0 … obs_{t-1}` and the commands, then map it into the 768-d observation manifold. No-future-leakage is mandatory and machine-checked: perturbing `obs_t` must leave the `cmd_{≤t}` prediction bit-identical (the per-genome guard scores any leak `−inf`). This is exactly the constraint that separates a world model from a smoother — the model must *predict* the next observation, never peek at it.

**The `(pred, h)` split** separates the readout (`pred`, scored by latent retrieval against same-verb foils) from the carried state (`h`, the substrate a downstream head or the next layer reads). Some archs make `pred` a per-position function of `h` (baseline Transformer, recency-ALiBi); others route a separate on-manifold read into `pred` that `h` never sees (the hippocampal and delta-rule archs). This distinction is load-bearing: a `head` wrapper that recomputes `pred = head(h)` silently bypasses any mechanism whose output does not flow through `h`.

## 2. Theoretical frameworks for sequence-architecture design

### 2.1 Self-attention Transformers and positional/recency priors

The Transformer [1] replaces recurrence with content-based self-attention: every position forms a query and retrieves a softmax-weighted combination of all positions' values, so any past token is *a priori* equally reachable and the model learns from data which distances matter. For a causal world model this is applied with an upper-triangular mask (future = `−inf`). Its strength is exact content-addressed recall over the whole visible history at O(L²) cost; its weakness — central to this project — is that the *learned, content-specific distance routing is exactly what must transfer to unseen systems*, and it is what tends not to.

Because raw self-attention is permutation-equivariant, position must be injected. The design axis matters here:
- **Absolute sinusoidal / learned embeddings** [1] mark opening tokens (e.g. the `uname`/config prefix that fixes system identity) but do not encode relative distance directly.
- **Relative position representations** [2] add learned biases indexed by `i−j` inside attention, letting the model reason about *distances* rather than absolute slots — apt when the same local `cd; ls` motif recurs at different absolute offsets.
- **Rotary position embedding (RoPE)** [3] rotates queries/keys by position-dependent angles so the dot-product depends on relative offset, with a natural decay of inter-token dependency with distance and compatibility with linear attention.
- **ALiBi** [4] adds a fixed non-positive linear penalty `−m_h·(i−j)` per head, biasing attention toward recent keys and enabling train-short/test-long extrapolation.

The project's first arch wins came precisely from encoding the *locality of a walk* as a prior. A shell exploration is a walk: the next observation depends far more on the local state (current directory, the file just `cd`'d into) than on distant tokens — except for the fixed system identity. `gh_recency_alibi_perhead.py` bakes this in as a *learned per-head* ALiBi slope (`−softplus(slope_h)·(i−j)`), so some heads become sharply local ("what is my cwd now") and some near-flat ("what system am I on"), with an optional undecayed head reserved for the identity path. It is a strict generalization of the baseline (flat slopes recover it) and numerically trivial (a bounded non-positive additive bias can never overflow a softmax). This, plus a broadcast system-identity feature, gave the first stackable ~+0.005–0.01 arch gains.

### 2.2 Recurrent, linear-attention, and state-space alternatives — the associative-memory view

The quadratic cost and the flat distance prior motivate a second family that carries a *bounded recurrent state* and updates it token-by-token, restoring O(L) inference. The unifying lens is that many of these are **associative memories** written and read online.

- **Linear attention** [5] drops the softmax, writing self-attention as a kernel dot-product and exploiting associativity so that attention becomes a running sum of key⊗value outer products — i.e. a matrix-valued memory `S_t = S_{t-1} + k_t v_tᵀ` read by `q_tᵀ S_t`. This makes a Transformer literally an RNN with a fast-weight memory, at linear cost.
- **Structured state-space models (S4)** [6] parameterize a linear continuous-time SSM with a structured (diagonal-plus-low-rank / HiPPO) transition, computed as a long convolution; they close much of the Transformer gap on very long sequences and dominate Long Range Arena.
- **Selective SSMs (Mamba)** [7] make the SSM parameters input-dependent (selective), recovering content-based reasoning — the ability to remember or ignore tokens based on their content — while keeping a hardware-aware linear scan. This is the property plain linear attention lacks.
- **RWKV** [8] and **RetNet** [9] are explicitly dual: they train in a parallel (attention-like) form and infer in an O(1) recurrent form; RetNet's *chunkwise* form interpolates — parallel within a chunk, recurrent across chunks — which is exactly the trick the champion arch reuses.
- **Parallelizable minimal RNNs (minGRU/minLSTM)** [10] show that stripping the hidden-state dependence out of the gates makes classic RNNs trainable by parallel scan and competitive with Mamba — evidence that much of the benefit is the *parallel-friendly linear recurrence*, not any one gating scheme. (The project's R9 round also produced a minGRU-style parallel-scan arch that beat the old champion at ~7× speed.)

The trade-off is memory capacity vs cost: a fixed-size recurrent state must *overwrite* to store new associations, whereas full attention keeps everything. This is the tension the delta rule addresses.

### 2.3 Fast weights and the delta rule (the project's central mechanism)

**Fast weights** are a second set of weights that change on the timescale of the input to store transient context, controlled by a slow network — Schmidhuber's 1992 fast-weight programmers, where one feedforward net emits context-dependent weight changes for another, storing temporal information in a single weight rather than a unit [11]; and Ba et al.'s 2016 fast-weight memory, an outer-product associative store that lets a net attend to the recent past, motivated as neurally plausible short-term synaptic plasticity [12].

The key modern connection: **linearized self-attention *is* a fast-weight programmer** [13]. Schlag, Irie & Schmidhuber show linear-attention's additive `S_t = S_{t-1} + k_t v_tᵀ` update is formally the 1992 mechanism, then identify its capacity flaw — purely additive writes saturate the memory — and fix it with a **delta-rule** write: before storing `(k_t, v_t)`, retrieve the current value `v̄ = S_{t-1}ᵀ k_t` and write only the *correction* `β_t (v_t − v̄) k_tᵀ`. This is the classic Widrow-Hoff delta rule applied online in memory space; it lets the store *edit* an existing key→value binding instead of piling onto it, sharply improving associative recall.

DeltaNet's obstacle was that the delta write is sequential (each step depends on the previous memory), unlike parallel attention. **Yang et al. 2024** [14] resolve this with a *chunkwise* algorithm: express a chunk of delta updates as a product of Householder-like transforms, reformulate it in the compact **WY representation**, and solve one lower-triangular system per chunk — recovering parallel training while computing the exact delta recurrence, and outperforming Mamba/GLA at 1.3B scale. **Gated DeltaNet** [15] adds a gated (decaying) memory so the store can also *erase*, combining Mamba-2's gating with the delta rule's targeted edits. **DeltaProduct** [16] builds on the interpretation of *DeltaNet's* single-step recurrence as *one step of online gradient descent per token on an associative-recall loss*, and generalizes it to `n_h` such steps per token (a product of `n_h` Householder transforms), interpolating between DeltaNet and dense transitions and improving state-tracking and length extrapolation — tying fast weights back to online learning.

This lineage is the spine of the project's arch progression. `r7_path_delta_fastweights_codex.py` writes each completed `(cmd, obs)` pair into **two** delta-rule memories — one keyed by command content, one by the recurrent path/context state — and reads them *in target space* at future command positions, so the prediction is an interpolation over *actual past observations* rather than a learned cmd→obs map that must transfer. `r9_chunked_delta_fastweights_codex.py` (the current champion) keeps that mechanism but replaces the token-by-token Python memory edits with the **chunkwise lower-triangular delta solve** on a small causal-Transformer trunk, plus a gated recency channel; it was both **faster (6.95×) and better** on both held-out splits — the WY/chunk reformulation of exactly the [14,15] mechanism. Its inline comments cite the DeltaNet chunkwise/WY and online-least-squares lineage and RLS/Kalman exponential forgetting as the write rule's control-theory analog.

### 2.4 Associative / Hopfield memory and the hippocampal lens

A parallel justification for content-addressable reads comes from **modern Hopfield networks** [17]: a continuous-state Hopfield net with an exponential energy stores exponentially many patterns and retrieves one in a single update — and its update rule is *equivalent to Transformer attention*. So "attention," "associative memory," and "pattern completion from a partial cue" are three names for one operation, which reframes the arch design question as *what to store, how to key it, and when to overwrite*.

The project's single largest arch jump came from taking this literally through a **neuroscience** lens. `r6_hippo_episodic_place_read.py` casts shell exploration as spatial navigation (`cd` moves between places, `ls`/`cat` are observations at a place) and imports two ideas:
- **Complementary Learning Systems** [19] / **Neural Episodic Control** [18]: a fast, content-addressable episodic store (hippocampus / a differentiable neural dictionary) binds `(context, observation)` in one shot and completes it later from a partial cue, complementing a slow neocortex (the Transformer) that only gradually learns statistical regularities. Here the slow map is exactly what fails to transfer to unseen systems; the fast per-sequence store needs *no* transfer — within one exploration a revisited place is completed one-shot from the real observation seen.
- **The Tolman-Eichenbaum Machine** [20]: generalization comes from *factorizing* a structural place/grid code from sensory content, so the place code indexes memory and transfers even when content differs. The arch's memory *key* is a factorized place code (causal command context + a projection of the command embedding), kept separate from the stored *value* (the raw observation embedding).

The mechanistic payoff is that the episodic read returns a *convex combination of actual past observation embeddings*, `e_t = Σ_{j<t} softmax(q_t·k_j) obs_j`, which is guaranteed to lie on the manifold that retrieval scores — unlike an arbitrary head output — and is confidence-gated so it fires only when memory is informative and otherwise relaxes to the parametric baseline. This is the same associative-memory operation as [17], but with the *value fixed to the on-manifold observation*, and it is the more attention-native realization of what the delta-rule archs later did with recurrent state.

### 2.5 World-model sequence backbones (context)

The broader model-based-RL literature independently converged on *predicting in latent space with a recurrent state model*. PlaNet's **Recurrent State-Space Model** [21] carries a deterministic + stochastic latent state, predicts forward purely in latent space, and plans by imagined latent rollouts — the same "predict latent consequences, then plan by latent distance" recipe this project tests, and a reminder that a world-model backbone must (a) integrate history into a compact state and (b) produce predictions in the space the objective scores. LeCun's JEPA program [22] frames the design goal: learn a predictor in a *joint embedding* space that captures dependencies without generating pixels/tokens, which is exactly why the arch predicts a frozen observation *embedding* rather than the literal output text.

## 3. Domain-specific findings — the literature meets this project's data

Three threads from the associative-memory literature explain *why* the neuro/delta-rule archs won here, beyond generic quality:

1. **On-manifold reads beat learned maps under distribution shift.** The retrieval metric scores squared-L2 to the true next observation against same-verb foils. A parametric head emits an arbitrary 768-d vector whose learned cmd→obs mapping may not transfer to an unseen image; a convex combination of *real past observation embeddings* [17,18] cannot leave the manifold and needs no cross-system transfer. Both the hippocampal read and the target-space delta read exploit this, which is consistent with them being the two biggest arch levers (0.5471 → 0.5607 → 0.5702 final margin).
2. **The delta rule's capacity edit matters more than raw recall.** Additive memories saturate [13]; the delta write edits an existing binding [13,14]. In a shell walk the same command (`ls /etc`) recurs and its correct observation must *overwrite* a stale guess, not average with it — the mechanistic reason the fast-weight archs edged past the attention-only episodic read.
3. **Chunkwise/WY parallelization is free quality-plus-speed when it is exact.** RetNet's chunk form [9] and DeltaNet's WY solve [14] show the recurrence can be computed exactly in a parallel-friendly block form. The champion's 6.95× speedup with a *gain* on both splits is a direct instance: same mechanism, hardware-aligned reformulation.

The project also records an important negative-methodological finding specific to arch search: **proxy→full direction is inconsistent across arch families.** R5/R8 arch proxies over-promised (clear proxy winners fell below the champion at full budget), while R9's chunkwise and minGRU archs were *under*-predicted by proxy. Unlike objective/perception chunks, arch mechanism changes interact with training-budget in family-specific ways, so an arch must be judged at full budget (4000 steps, 3 seeds, inner + untouched final) and mechanism-level plausibility is a hypothesis generator, not a verdict. Champion validations remained strict throughout: the R9 arch clears gen-twin (0.884 vs 0.572) and history-ablation (0.836 vs 0.441) on the untouched final-test.

## 4. Design guidance & open questions for future `arch` impls

- **Prefer reads whose values are on the observation manifold.** The strongest archs route a convex combination / associative read of *raw past observation embeddings* into `pred`, not just a learned projection of `h`. Keep such a read gated so it degrades to the parametric baseline when memory is uninformative.
- **Factorize key from value.** Separate the *place/context/content key* (learned, structural, transferable [20]) from the stored *observation value* (on-manifold). Multiple keys (content key + path key) into a shared value store paid off (R7/R9).
- **Use the delta rule, not additive writes, and prefer erase-capable (gated) variants** [13,14,15] so recurring commands overwrite stale bindings.
- **Chunkwise/WY or parallel-scan formulations are the default** for any recurrent-memory arch: they are exact, parallel-trainable, and — as R9 showed — can be faster *and* better [9,14]. Verify exactness by state_dict-identical forward equivalence when reformulating for speed.
- **Bake in the walk's locality as a prior, cheaply.** Learned per-head recency (ALiBi-style) [4] with a reserved undecayed identity head is a strict, numerically trivial generalization of the baseline; it stacks with the memory mechanisms.
- **Numerical stability is a first-class arch property.** Avoid exponentials over sequence position and division by accumulated mass (the diverged EMA-bank arch); a 15-step real-data integration train — not just a single-forward smoke — is required to catch NaNs.

**Open questions.** (1) Can a *selective* SSM [7] or gated DeltaNet [15] backbone subsume both the recency prior and the associative read in one linear-time primitive, removing the Transformer trunk entirely? R9's minGRU variant is a start. (2) Is there a principled *multi-step* write (DeltaProduct [16]) that helps state-tracking across a long exploration without the budget-sensitivity that sank the RLS/Kaczmarz DeltaProduct write at full budget? (3) The R10/R11 planning rounds found the frozen substrate's off-manifold prediction norms limit latent-MPC; can an arch be designed whose reads are *calibrated in norm* by construction (the on-manifold read is a lever the parametric head lacks)? (4) Does a `head` that is a per-position function of `h` ever recover the on-manifold-read gains, or is routing a separate read into `pred` fundamental?
